Controlling the operation of Multiflash calculations

The operation of the Multiflash interface can be controlled with a number of options:

mass

Use mass basis for component inputs and outputs.

noderiv

Disable the use and calculation of analytical derivatives.

noexptransform

Disable the 'exponential transform' applied to inputs. The exponential transform prevents that negative values are sent to Multiflash as inputs for calculations and performs a smooth, exponential transformation which projects very small negative values to positive values. This is particularly useful for very small concentrations which may become negative depending on the chosen bounds due to the numerical accuracy.

In order to be smooth the transformation is already applied at very small positive values which may lead to undesired results under certain circumstances. In such a case the transformation can be turned off and it is strictly required to ensure with proper bounds that only feasible values are sent to Multiflash.

components

Create a phys props instance which can only retrieve component names from the specified command file. No Multiflash license is requested, unlike for all other configurations. Any methods called other than "COMPONENTS" or "NUMBEROFCOMPONENTS" will fail.

usehotstart
Enables hot-starting of flash calculations. Previous results are used as the initial guess of a subsequent flash iterations inside the Multiflash engine which normally leads to shorter computation times. In rare situations this may lead to marginally different flash results which may cause failures in the overall activity execution; such a difference warrants an investigation whether the model operates in a non-robust region.

The general syntax for specifying options is

PhysProp := "Multiflash::<option_1>:<...>:<option_N>:<command file name>" ;
and an example with multiple options would look like this:
PhysProp := "Multiflash::mass:noderiv:AlkaneMixture.mfl" ;
