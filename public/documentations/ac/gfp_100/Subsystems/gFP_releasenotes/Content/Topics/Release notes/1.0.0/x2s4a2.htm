Upgrading legacy flowsheets to use gPROMS FormulatedProducts

For flowsheets containing models that have been migrated to gFP, it is possible to upgrade the flowsheet to use the new libraries. Currently, due to limited backwards compatibility, this is a manual task and can be completed following the steps below:

  1. Open the gFP libraries and the required legacy libraries
  2. Open the target flowsheet and save a copy as "<flowsheet name> - gFP"
    1. Keep both copies open
    2. Only edit the newly saved project
  3. In the newly saved project add the required cross-references by either:
    1. Dragging on and then deleting the unit operations required from the gFP libraries
    2. Editing the cross-references of the project directly (right-click the project -> Cross-references); add the gFP libraries required
  4. Right-click each model (other than global specifications) in turn and in the context menu select "Change model..."
  5. In the pop-up dialog, select "ALL MODELs" radio button and find the required model in the drop-down and press OK
  6. All models should now be converted to gFP models
    1. Remove legacy cross-references from the project (right-click the project -> Cross-references)
    2. Ensure the corresponding legacy library gFP version fundamentals library is cross-referenced; this will maintain your global specifications e.g. if using gSOLIDS, "3 gSOLIDS Fundamentals" should be in the cross-references
  7. Optional: Upgrade to gFORMULATE global specifications to use the new physical properties database. This requires using the Data Import and Properties Manager interface to import the required species into the database as well as configuring a material systems file.
  8. Open the original project and upgraded version side by side; compare dialog specifications for each model and transfer any that are missing or changed values
  9. Close the original project and the legacy libraries.
  10. Use the new project as usual.
