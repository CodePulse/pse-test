Case 7: Model level parametric case study

High fidelity models can reduce experimental work to a large extent. The simulation of high fidelity models enables safe and efficient experimentation to obtain solutions to a number of 'what if' problems. GSA can further improve this efficiency by simulating multiple scenarios with a single GSA execution.

The purpose of this example is to investigate the variation of yield and temperature as a result of increasing the inlet molar fraction of methanol from 0.1 to 0.9.

Create a GSA entity named 'Case7_ParametricCaseStudy'. The settings for the General tab are shown in the figure below. In this example, uncertainty analysis is performed with the quasi-random Sobol sample generation method.

Case 7: General tab

Specify the model level parameters of interest as factors in the Factors tab. Either use the 'Select...' button or enter the full variable path into the 'Variable in model' column, pressing Control+Space to use autocompletion.

Select each factor and click on the 'Edit...' button to specify all other factor settings. Note that, in a 'Gridded set' distribution, the number of intervals does not equal the total number of sample points of the factor (number of sample points = number of intervals + 1).

Case 7: Editing factors

As there are no sampling restrictions in this example, leave section 'Sampling restrictions' blank.

Specify the number of uncertainty scenarios in the 'Number of samples' section. In the absence of uncertainty scenarios, leave it at the default value of one. The total number of realisations of the base activity to be calculated is given by: 

Notation

  Nd Number of deterministic cases
  N Number of uncertainty scenarios (user defined) 

Specify the responses in the Responses tab. Similar to factors, either use the 'Select...' button or enter the full variable path directly into the 'Variable in model' column, pressing Control+Space to use autocompletion.

Case 7: Responses tab

The 'GSA language' tab is automatically updated once the General, Factors and Responses tabs are set up. Launch the execution by right-clicking on the GSA entity and selecting 'Analyse system' or pressing icon in the toolbar. 

Case 7: Executing GSA

The 'Analyse System' dialog will be displayed. Change the Case name and select the desired number of workers for parallel computation. Note that this number should not exceed the total number of logical cores on the machine.

Once execution completes successfully, right-click on the 'System Analysis Group' folder and select 'New result element...' to create result elements. Alternatively, you can copy and paste result elements from other Cases. Note that the 'Factor Sensitivity Table' is not applicable in the context of uncertainty analysis and will not be populated with results.

Case 7: Creating a new result element

The 'Scatter 2d' result element can be used to illustrate the relation between any two factors, responses or a factor and a response. Select which variables to show on the X and Y axes via the Attributes, shown in the figure below.
