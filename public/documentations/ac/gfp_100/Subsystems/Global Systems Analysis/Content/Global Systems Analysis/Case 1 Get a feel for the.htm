Case 1: Understanding the model

A gPROMS model typically has some degrees of freedom which are satisfied by assigning values to variables. Different assigned values typically result in different model behavior. To assess the robustness of a model or gain insight into its behaviour, it is possible to obtain the results of a number of simulations at the end of a single GSA activity execution.

This case illustrates how to perform uncertainty and sensitivity analysis using assigned variables as factors and key performance indicators (KPIs) as responses.

Create a GSA entity named 'Case1_Sensitivity' and observe that it contains five tabs: General, Factors, Responses, 'GSA language', and Properties.

In the General tab, specify the 'Base activity for analysis', 'Type of analysis to be performed', 'Method to be used for analysis' and 'Sample generation method' as shown in the figure below. In this example, variance-based sensitivity analysis is performed using the quasi-random Sobol sample generation method.

After defining each factor, click on the 'Edit...' button to specify its variability/probability distribution and other information, as shown in the figure below. To assess model robustness and better understand the behavior of the model, select 'Uniform distribution' to ensure that the entire range of the factor space will be covered. In this example, five factors are specified, all of which assumed to have a uniform distribution.

Note that the 'Sampling restrictions' section is only available for uncertainty analysis at present.

In section 'Number of samples' specify the desired number of uncertainty scenarios. The total number of realisations of the base activity to be calculated is given by: 

Notation

  Nd Number of deterministic cases
  N Number of uncertainty scenarios (user defined)
  K Number of probabilistic factors

Specify the responses in the Responses tab. Similar to factors, it is possible to either use the 'Select...' button or enter the full variable path directly into the 'Variable in model' column, pressing Control+Space to use auto-completion.

Case 1: Responses tab

The 'GSA language' tab is automatically updated once tabs General, Factors and Responses are set up. Launch the execution by right-clicking on the GSA entity and selecting 'Analyse system...' or by pressing icon in the toolbar. 

Case 1: Executing GSA

The 'Analyse System' dialog will be displayed. It is possible to change the Case name and select the desired number of workers for parallel computation. Note that this number should not exceed the total number of logical cores on the machine.

Once execution completes successfully, right-click on the 'System Analysis Group' folder within the Case and select 'New result element...' to create result elements.

There are four options available: 'Distribution Statistics Table', 'Factor Sensitivity Table', Histogram, and 'Scatter 2d'. Result elements can also be copied and pasted to and from Projects and Cases.

Case 1: Creating a result element

The 'Factor Sensitivity Table' indicates that the total effect of inlet molar fraction 1, on response yield, is much greater than that of any other factor. The total effects of coolant temperature, inlet molar flow rate and inlet molar fraction 1 on temperature are much greater than those of other factors. Among these three, inlet molar fraction 1 is the most important factor. In contrast, pressure has no effect on any of the responses and can be ignored in subsequent analyses, provided that the responses of interest are the same.
