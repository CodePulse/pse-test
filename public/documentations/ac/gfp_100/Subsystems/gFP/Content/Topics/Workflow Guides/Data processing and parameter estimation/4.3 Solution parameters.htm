Solution parameters

Adjusting the output level in the SOLUTIONPARAMETERS section of the Process will enable you to change the amount of information sent to the execution window during the execution of an Estimation activity.  In order to do this it is recommended to follow the points below:

  1. Right click on the process entity associated with the parameter estimation entity
  2. Go to Open on tab>Solution parameters
  3. Go to Numerical solvers>PESolver.  Change the OutputLevel to 4.
  4. Go to Numerical solvers>PESolver >MINLPSolver.  Change the OutputLevel to 2.

After adjusting the solution parameters, one is ready to execute the parameter estimation entity.  In order to do this, right click on the parameter estimation entity you created and then click on “Estimate”. Alternatively, the estimate icon, , located on the toolbar can be used to run the parameter estimation entity. If there are any syntactical, cross-referencing mistakes or other mistakes in the setup, these will be reported.  Otherwise, an execution options dialog will be displayed, similar to that shown below. 

 

Execution options dialog.

Concept > Monitoring progress of parameter estimation. The progress of the parameter estimation run can be monitored through the trajectories. As the parameter estimation optimiser steps through each iteration to obtain the best fit parameters, the user can view (graphically) the progress via a comparison of “current” prediction versus measurements. Due to the complexity of the problem, large parameter estimation runs may take several hours to finish.

 

Best practice> Graphical monitoring through trajectory plots with measured data is a good way to check if the initial values of the parameters are adequate and also if the trends and variances in the measurements are as expected. Many times, it is necessary to abort the run, make a change and re-run.

 

In that dialog, ensure that the dropdown field below the Reporting interval field is set to Major iterations.  That option allows monitoring the progress of the parameter estimation in terms of the quality of fit via the Trajectories in the Case that will be created after clicking OK in the dialog above, an example is shown below.

Once the parameter estimation has been successfully performed:

  • View the results in the case.
  • Export the results to Excel by double clicking on the stat-mr entity in the Results folder of the Case.
  • Copy the estimated values back to the relevant unit operation model in the original flowsheet model to perform a comprehensive simulation.
