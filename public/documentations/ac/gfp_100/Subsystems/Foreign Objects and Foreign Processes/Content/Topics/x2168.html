Automatic closing of Microsoft Excel workbooks

If Microsoft Excel workbooks (.xls or .xlsx) are stored in a Project, a conflict will arise when a gPROMS Activity is executed multiple times because gPROMS tries to open several copies of the (same) workbook with identical names one after the other. Microsoft Excel does not permit opening several workbooks with the same name and therefore issues an error message.

gPROMS offers an option to automatically close workbooks at the end of an Activity execution if a flag called autoclose is added to the FO specification, see the example below.

SET
    ParameterPath       := "ExcelFO::autoclose:ForeignObjectValue" ;

A workbook will only be closed automatically if it has been opened by gPROMS and not by the user prior to the Activity execution.

Using this flag is not always useful, for instance not when data need to be visualised after an Activity execution has finished. In these cases it is advisable to close the workbook manually or to have the workbook control when the execution is finished.
