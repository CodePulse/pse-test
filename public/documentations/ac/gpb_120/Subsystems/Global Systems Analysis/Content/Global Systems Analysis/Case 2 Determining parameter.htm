Case 2: Determining parameter importance

If there are limited resources to improve parameter estimates, identify which parameters most influence the KPIs. GSA can be used to sort the importance of parameters for each response. 

GSA entity 'Case2_ParaImportance' demonstrates how to set up such an analysis. The necessary settings for the General tab are shown in the figure below.

Case 2: General tab

Specify the parameters of interest as factors in the Factors tab. Either use the 'Select...' button or enter the full variable path into the 'Variable in model' column, pressing Control+Space to use autocompletion.

Select each factor and click on the 'Edit...' button to specify all other factor settings. Note that, since only variables are eligible as factors, all parameters of interest must be moved from the PARAMETER section to the VARIABLE section of the model before the GSA entity is set up.

Note that the 'Sampling restrictions' section is only available for uncertainty analysis at present.

In section 'Number of samples', specify the number of uncertainty scenarios. The total number of realisations of the base activity to be calculated is given by: 

Notation

  Nd Number of deterministic cases
  N Number of uncertainty scenarios (user defined)
  K Number of probabilistic factors

Case 2: Factors tab

Specify the responses in the Responses tab. Similar to factors, either use the 'Select...' button or enter the full variable path directly into the 'Variable in model' column, pressing Control+Space to use autocompletion.

Case 2: Responses tab

The 'GSA language' tab is automatically updated once tabs General, Factors and Responses are set up. Launch execution by right-clicking the GSA entity and selecting 'Analyse system' or pressing icon in the toolbar. 

Case 2: Executing GSA

The 'Analyse System' dialog will be displayed. Change the Case name and select the desired number of workers for parallel computation. Note that this number should not exceed the total number of logical cores on the machine.

Once execution completes successfully, right-click on the 'System Analysis Group' group and select 'New result element...' to create result elements. Alternatively, you can copy and paste the result elements from Case 1.

Case 2: Creating a result element

The 'Factor Sensitivity Table' indicates that the reaction activation energy and reaction pre-exponential coefficient for reaction 1 are much more important than those for reaction 2. Therefore, reaction 1 is dominant, and reaction 2 should be ignored in subsequent analyses. It can also be inferred that the pre-exponential coefficients are more relevant than the activation energies for the responses of interest. This means that the best way to reduce uncertainty in the responses would be to focus on the pre-exponential coefficients.
