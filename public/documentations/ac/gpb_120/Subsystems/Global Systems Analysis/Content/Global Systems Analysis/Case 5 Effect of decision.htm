Case 5: Effect of decision variable uncertainty on the objective function and constraints

After executing an optimisation activity, it is important to understand the effect of uncertainties in decision variables on the objective function and constraints. For example, since the coolant temperature cannot be perfectly controlled, variations are inevitable in practice. GSA can be used to see how much this variability affects the objective function and constraint values.

GSA entity 'Case5_DecVarUncer' demonstrates how to set up such an analysis. The settings for the General tab are shown in the figure below. In this example, uncertainty analysis is performed with the quasi-random Sobol sample generation method.

Case 5: General tab

Specify the decision variables as factors in the Factors tab. Either use the 'Select...' button or enter the full variable path into the 'Variable in model' column, pressing Control+Space to use autocompletion. This example assumes all decision variables follow a normal distribution, since it is expected that their values vary around a mean.

Add factor constraints in the 'Sampling restrictions' section. In this example, it is assumed that the inlet flow is cooled by a heat exchanger before going into the CSTR. If the flow rate increases, the inlet temperature will also increase due to the constant heat duty from the heat exchanger, whereas if the flow rate decreases, the inlet temperature will also decrease. Therefore, there is a correlation between the inlet temperature and the inlet flow rate.

Click on the 'Add...' button and enter the constraint in the 'Constraint expression' field as shown in the figure below. There are two sampling restrictions in this example.

The finished Factors tab should resemble the one shown in the figure below.

Case 5: Factors tab

Specify the responses in the Responses tab. Similar to factors, either use the Select button or enter the full variable path directly into the 'Variable in model column', pressing Control+Space to use autocompletion.

Case 5: Responses tab

The 'GSA language' tab is automatically updated once tabs General, Factors and Responses are set up. Launch the execution by right-clicking on the GSA entity and selecting 'Analyse system' or pressing icon in the toolbar. 

Case 5: Executing GSA

The 'Analyse System' dialog will be displayed. It is possible to change the Case name and select the desired number of workers for parallel computation. Note that this number should not exceed the total number of logical cores on the machine.

Once execution completes successfully, right-click on the 'System Analysis Group' folder and select 'New result element...' to create result elements. Alternatively, copy and paste result elements from other Cases.

Case 5: Creating a new result element

The 'Scatter 2d' result element can be used to illustrate the relation between any two factors, responses or a factor and a response. Select which variables to show on the X and Y axes via the Attributes, shown in the figure below. By plotting the inlet temperature on the Y-axis against the inlet molar flow rate on the X-axis, it is possible to observe the restrictions between the two factors.

Case 5: 'Scatter 2d' plot of inlet temperature against inlet molar flow rate
