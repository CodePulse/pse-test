Making Model specifications

Specifications are made using Model specification dialogs that appear when double clicking a component Model icon on the Topology tab of a composite Model. Giving specifications using these dialogs corresponds to Setting and Assigning values to the gPROMS Parameters, Variables and Selectors declared as part of this component Model1See the Custom Modelling Guide for more details on which gPROMS quantities need Assigning, how these specifications are classified and for construction and customisation of dialog boxes.

A dialog box of a reactor on the “Initial conditions” tab

Specification groups
Specification groups allow the Model user to select Model modes and choose from different specification sets. The Reactor example in the figure above contains three group selectors, determining the specification mode [Standard, Advanced], the heat input mode [Controlled, Manual] and the mode of initial conditions, see the expanded list in the figure. Changing the group may change the number of allowable specifications available to the user.
Specifications tabs
Specifications tabs can be used to segregate specifications logically. In the Reactor example, initial conditions have their own tab - these specifications only apply at the start of the simulation, whereas the configuration and reaction data specifications apply throughout the time horizon.
A dialog box may have 0, 1 or more tabs depending upon its design.
Specification properties
Specification properties that are mandatory have greyed-out check boxes (temperature and pressure in the example) whereas those that are optional have enabled check boxes. In many cases, the model developer will have ensured the default number of specifications satisfy the degrees of freedom to ensure successful simulation.
Specifications that involve array quantities appear as tables with the array dimension labelled vertically up the left hand side.
Specification bounds
Specification bounds on certain specifications may have been imposed by the Model developer. If values are entered outside these bounds, the border turns red indicating that an illegal specification has been made. Hovering the mouse pointer over the values themselves should display the bounds in another tool tip.

Units of measurement
Units of measurement are displayed to the right of numerical specifications. It depends on the model developer if they are fixed or if choices are available. In the figures above, all units of measurement are fixed. The figure below displays selectable choices.

Selectable units of measurement

If units of measurement are selectable, they are displayed in combo-boxes in contrast to fixed labels for unmodifiable ones. Changing the unit of measurement will automatically convert the numerical value so that it conforms to the new choice. In the example above, changing to Kelvin will display the numerical value as 300.
Dialog box buttons

OK - confirms changes made to specifications

Cancel - cancels any changes made

Reset all - returns all specifications to their default values

Help - brings up the help for that Model
