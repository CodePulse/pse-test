Writing Output Channels in FORTRAN and C

As described in the section on the gPROMS Output Channel Interface, any Output Channel implementation has to provide six communication procedures and two procedures for initialisation and termination respectively. For convenience, skeleton files are provided for both FORTRAN and C. The skeletons only contain the calling frame for the procedures. The name of these files are oci_demo_c.c and oci_demo_f.f. The C version also relies on the header files gOCInterface.h and gTypes.h which

  • provide prototype definitions for the eight Output Channel Interface routines,

  • define a macro which ensures that the function prototypes are exported when compiling for Windows, and

  • define own versions of the basic types for ease of portability.

Please note: It is strongly recommended that you do not change the prototypes and type definitions because they ensure that the function calls follow the conventions gPROMS uses internally.
