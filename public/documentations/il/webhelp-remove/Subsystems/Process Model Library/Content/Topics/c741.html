Guidelines for constructing PML flowsheet Models

The PML is designed to enable the user to quickly construct process flowsheet models. When building flowsheets, the following general guidelines are suggested:

  1. Set up a working project - create a new flowsheet Model entity in a working project: this should be set-up to cross reference all of the PML library projects. Do not copy the PML Models into the working project unless it is necessary to modify a model.

  2. Construct a Model topology - a flowsheet model is constructed by dragging and dropping PML Models from the project tree (or icons from the model palette) on to its Model topology tab.

    • When defining the topology of the flowsheet model, remember that all PML material ports must have at least a single connection (otherwise the model will be underspecified). Many PML Node models support array ports which allow multiple connections to be made to the same Port.

    • A material stream must always start at a PML Source Model and terminate with a PML Sink Model1Potentially, a dead-end model may also be used to terminate a material stream.Consequently, all flowsheets can be expected to have at least one instance of a Source model and one instance of a Sink model.

  3. Provide Model Specifications - double click on each Model's icon to activate its Specification dialog:

    • Scalar quantities have default values which can be over-ridden

    • Array quantities such as mass fraction arrays do not have default values and you must take care to provide the appropriate number of values

    • Most numeric quantities have bounds to ensure that sensible values are provided

To execute the simulation you will need to create a Process Entity. Once associated with a particular Model entity it will automatically contain all the specifications that have been provided via Specification dialogs.
