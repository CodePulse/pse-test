Flowsheeting diagnostics

The Flowsheet Diagnostics Panel provides instantaneous feedback regarding modelling issues before the execution of a gPROMS Activity and it appears below a flowsheet if there are any warnings or errors.

Flowsheet Diagnostics Panel

The diagnostics panel by default displays errors and warnings related to the units on the flowsheet. They concern, for instance, the following:

  • missing (obligatory or optional-on) specifications
  • erroneous specifications where bounds are not observed or entries cannot be recognised
  • connectivity errors
  • and more.

The panel is updated in real-time which means that any issue resolution but also any newly introduced issues are immediately reflected in the panel’s messages. In order to make the issue resolution easy accessible, a double-click on a message navigates to the source of the issue.

ModelBuilder preferences allow configuring which types of messages to display and how, see the figure below. All options un-selected is the default.

Flowsheet validation preferences

The available choices determine

  • if units that use pre-defined default values for some of the specifications are reported; checking this is very useful when a user wishes to ascertain that all specifications of the units on a flowsheet have been visited; note that when entering a value which is identical to a default, it counts as an explicit specification
  • if hidden unit attributes are validated that an end user normally does see because the model developer has chosen to conceal them
  • if errors in the model of a unit are also reported in this panel; this is useful if an end user wishes to see model errors reported also on the flowsheet and not only in the library project where the model is defined
  • how units are decorated: with error or warning icon overlays as depicted in the screenshot above, or as coloured boxes around the unit icon.
