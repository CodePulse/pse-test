Scenario or parametric analysis

In a scenario or parametric analysis the impact of factor variability on a response is determined. This is achieved by executing simulations for different factor values and investigating their effect on the response. Although this can be achieved by executing several simulations manually, using GSA provides some advantages:

  • It is easier to set up a large number of simulations to cover a multi-gridded space.
  • Failed simulations do not cause GSA execution to stop (as long as the threshold of failed simulations is not exceeded). Instead, it will continue until the requested sample space is covered and display the grids for which it was able to find a solution.
  • It is easier to analyse the results since all data are available in a single location.
  • If access to a gPROMS High Performance Computing (HPC) license is available, GSA can execute the simulations in parallel at the expense of a single set of licenses instead of requesting a set for each simulation launched manually.

Scenario or parametric analysis commonly relies on deterministic scenarios and/or changes in factor values. This type of analysis can be performed by setting up a GSA entity for uncertainty analysis and defining all factors as deterministic, that is, they have known values. This is the most straightforward type of analysis.
