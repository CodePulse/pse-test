Choosing variables

In the section on model requirements it has been explained that model variables need to be of a specific type depending on the role that they play in model validation. Controls, for instance, have to be Assigned variables. Identifying the type requires the inspection of the corresponding sections of Processes and Models and to use the assistance of gPROMS' efficient search capabilities.

On the other hand, Models that are equipped with a Public Model Interface and use Model specification dialogs allow navigating and selecting their variables using graphical editors. The permissible choices that are presented in these editors depend on the roles and are limited by the required type.

Variable Type Permissible Choice
Parameters to be estimated Assigned variable
Experimental controls Assigned variable
Experimental initial conditions Initial conditions
Measured variables Calculated variable
Constraints Calculated variable

The choice editors not only limit the choice to admissible entries but also allow an easy navigation through a flowsheet and the different tabs of Model specification dialogs. The functionality of the selection dialog is illustrated at the hand of the flowsheet of the standard example PML Batch Reactor where kinetic parameters of reactions in the central reactor are to be estimated, see the figure below.

Flowsheet of PML Batch Reactor example

The parameters Arrhenius constant and Arrhenius activation energy are part of the reactor's Model specification dialog and are shown on the second tab called Reaction Data.

Reactor model specification dialog

When it comes to choosing parameters to be estimated in Parameter Estimation or in Experiment Design, a dialog such as the one in the following picture is displayed which allows to choose the desired variable by navigating through a well-structured tree comprising all units on a flowsheet. Since Assigned variables need to be presented that are included as specifications on a Model specification dialog, the dialog also sorts the permissible choices by dialog tab.

Selection of parameters to be estimated

The available choices can be narrowed down while typing by using the instant search field in the upper right corner, which can considerably reduce the complexity. There are further "Advanced search options".

Choices narrowed down while typing by using the instant search field

The dialog for selecting initial conditions looks similar, the only difference being the content because the permissible choices here are narrowed down to variables that are marked in the Public Model Interface to appear in the Initial section of a Process. As can be seen, only entries from the tab Initial Conditions are available.

Selection of initial conditions

Calculated variables, on the other hand, can be part of the Public Model Interface but are not part of any dialog specification. As is shown in the following example choice of Measured variables, the flowsheet structure is still represented but specification tabs do not appear.

Selection of measured variables

The result of these choices is then displayed in the corresponding editors. The choice of Initial Conditions, for instance, is summarised in the General tab of the Parameter Estimation editor using the descriptive names of the chosen variables. How the full gPROMS path is accessible is explained in the section on direct editing of variable paths.

Note

Depending on the application preferences, it may not be enabled to view or set the full gPROMS variable path.
