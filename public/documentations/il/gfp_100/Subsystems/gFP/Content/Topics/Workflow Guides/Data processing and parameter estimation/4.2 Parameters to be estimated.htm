Parameters to be estimated

The Parameters to be estimated are a subset of the ASSIGNed variables in the Process entity, which will change the model predictions if they are adjusted.

 

Parameter estimation entity - Parameters to be estimated tab.

 

“Parameters to be estimated”, see Figure above, can be selected by clicking “Select…” in the right hand corner. The “Parameters to be estimated” dialog that appears will provide a list of the possible parameters available within a given process, like that shown below. This dialog also provides the current specified value within the model flowsheet and the units for each available parameter. For example, “Total heat loss” has units of J/s and is currently specified a value of 0 J/s within the dialog of the “fluid bed dryer agglomerator” model, as shown below. 

 

Select parameters to be estimated dialog.

 

The final step is to give reasonable bounds to the estimated parameters and also their initial guess values.

 

Once the above steps are complete, the parameter estimation entity is ready.

 

Best practice> Initial guesses and scaling for parameter estimation.  It is important to start with “reasonable” initial values for the parameters to be estimated. To do this, you may start with the default values and adjust parameters in simulation mode, if needed. Also, if the values of the parameters differ considerably (as is the case here), it is recommended to use a scaling setting of 1 (Process Entity > Solution Parameters > PESolver > MINLPSolver > Scaling). This scales the parameters using their lower and upper bounds and results in a more efficient and accurate solution.
