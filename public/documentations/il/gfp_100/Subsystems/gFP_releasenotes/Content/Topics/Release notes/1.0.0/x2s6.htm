Converting custom models for kinetics and properties

Custom modelling kernels built for inclusion with previous versions of gPROMS platform software will need to be migrated to the new Custom model templates provided in gFP 1.0 for access by the gFP 1.0 library models

  • First, load both the new gFP 1.0 libraries and the legacy model libraries, you should close all other editor and flowsheet windows by selecting Window>Close all
  • Close the gFORMULATE Custom library in the project tree. The custom kernel project can be found under "Examples" -> "Custom modelling templates" -> "Kinetic and property model templates" -> "0 gFORMULATE Custom.gPJ-FP". Open the project and copy the new gFP 1.0 custom kernel model template (actual name dependent on kernel of interest) to your project.
  • Next, load only the legacy custom model kernel
  • Select Window>Show side by side
    • You can also select both the new template and legacy custom kernel and utilize the Compare function to ascertain the extent of the differences
  • You can use copy/paste to migrate the user defined portions of the code from old to new
    • In some cases, the complete path or variable/parameter names may have changed
      • Many times these changes are obvious and should be corrected by you
      • When these changes are not obvious, or if reasonable efforts to affect the desired changes have failed, you can email the project in question to the Formulated Products support queue - support.formulatedproducts@psenterprise.com
  • Once the changes have been made to the custom kernel templates, you should open the affected unit operations in the project via the flowsheet to determine if the desired changes have been effected in the unit operation GUIs and that the correct user defined parameters have been entered
