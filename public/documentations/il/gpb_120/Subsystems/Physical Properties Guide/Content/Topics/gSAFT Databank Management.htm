gSAFT Databank Management

Using a custom databank

The distributed databanks (VR_SW_databank.dtb and GC_Mie_databank.dtb) are provided in encrypted form, and their contents can neither be viewed nor modified. A separately licensed feature for databank management allows users to use the gSAFT foreign object with non-encrypted custom databanks. A non-encrypted databank can be used to initialise the gSAFT foreign object in the same fashion as an encrypted one.

gSAFT::non_encrypted_databank.xml<compound1,compound2,compound3,…>

Note that the .xml extension is used for non-encrypted databanks.

When a custom databank is to be used for initialising the gSAFT foreign object, it needs to be attached to the project. This can be done by importing the file, by selecting Import files… from the Tools menu, as shown in the image below:

Importing a custom databank

Note that the option All Files in the file selection dialog needs to be selected to view all the available files that can be imported from a given location.

Once imported, the file should appear under the Miscellaneous Files in the project tree, as shown below:

The imported custom databank appears under the Miscellaneous Files entity of the project tree

Creating a custom gSAFT databank

A gSAFT databank is required to have a <gsaft> element as the root element, where the equation of state (EOS) that the databank corresponds to is specified. This is done by specifying the value of the theory attribute of the <gsaft> element. The valid values for this attribute are "vr_sw" and "gc_mie", for specifying SAFT-VR SW and SAFT-g Mie, respectively. An example of a gSAFT databank that specifies the SAFT-VR SW EoS to be used is shown below: 

<gsaft theory="vr_sw">
...					
</gsaft>

For more information on how to create a custom databank see the relevant sections for:

  • SAFT-VR SW or
  • SAFT-γ Mie.
