Case 4: Selecting decision variables for optimisation

Process models can be used to maximize production and minimise cost, among others. In an optimisation problem, the decision variables which affect the objective function and constraints must be determined first. GSA can be used to identify which variables influence the objective function or constraints the most facilitating the selection of decision variables.

In this example, optimisation entity 'Opt_Yield' is set up to maximize the yield of methyl acetate. The only constraint specifies that the temperature must be maintained within a certain range (see the explanation of end-point constraints in the 'Optimisation Guide' and the optimisation entity 'Opt_Yield' in the GSA example Project for details).

GSA entity 'Case4_DecVarSelection' demonstrates how to set up such an analysis. The settings for the General tab are shown in the figure below. In this example, sensitivity analysis is performed using the quasi-random Sobol sample generation method.

Case 4: General tab

Specify the variables of interest as factors in the Factors tab. Either use the 'Select...' button or enter the full variable path into the 'Variable in model' column, pressing Control+Space to use autocompletion.

Select each factor and click on the 'Edit...' button to specify all other factor settings. In this example, since the effects of decision variables on both objective function and constraint will be examined, a uniform distribution is the most appropriate.

Note that the 'Sampling restrictions' section is only available for uncertainty analysis at present.

Specify the number of uncertainty scenarios in section 'Number of samples'. The total number of realisations of the base activity to be calculated is given by: 

Notation

  Nd Number of deterministic cases
  N Number of uncertainty scenarios (user defined)
  K Number of probabilistic factors

The finished Factors tab should resemble the one shown in the figure below.

Case 4: Factors tab

Specify the responses in the Responses tab. Similar to factors, either use the 'Select...' button or enter the full variable path directly into the 'Variable in model column', pressing Control+Space to use autocompletion.

Case 4: Responses tab

The 'GSA language' tab is automatically updated once tabs General, Factors and Responses are set up. Launch the execution by right-clicking the GSA entity and selecting 'Analyse system' or pressing icon in the toolbar. 

Case 4: Executing GSA

The 'Analyse System' dialog will be displayed. It is possible to change the Case name and select the desired number of workers for parallel computation. Note that this number should not exceed the total number of logical cores on the machine.

Once execution completes successfully, right-click on the 'System Analysis Group' folder and select 'New result element...' to create result elements. Alternatively, copy and paste result elements from other Cases.

Case 4: Creating a result element

In this example, the 'Factor Sensitivity Table' provides the most relevant information. It is clear that the inlet molar fractions are the factors which most influence the yield. Although the coolant temperature and inlet molar flow rate also affect the yield, their effect is not significant in comparison to that of the inlet molar fractions. The effect of inlet temperature on the reactor temperature is negligible and can be ignored when compared with other factors. With respect to the constraint on the reactor temperature, the inlet molar fraction is the most important factor, although the inlet molar flow rate and coolant temperature also have some influence.
