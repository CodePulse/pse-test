Management of Cases

Once the user presses the OK button in the execution control dialog, gPROMS ProcessBuilder creates the Case. Just like a Project, a Case appears as a sub-tree of gPROMS ProcessBuilder's navigation tree. However, unlike most Projects, all entries in a Case are read-only, and this is indicated by a lock symbol annotating each entry in the Case sub-tree.

Read-only Cases

Cases may be saved to disk using standard Save and "Save as..." mechanisms from gPROMS ProcessBuilder's Project menu. The files used for their permanent storage have the .gCS suffix and are marked as read only. Cases may also be closed (with or without saving) at any time, and also be opened at a later stage using standard Project-management mechanisms.

Cases and activity execution

The default choices for the entries in the execution control dialog can be specified as part of the user's gPROMS ProcessBuilder Preferences. Further configuration options can be accessed by highlighting the "Activity execution defaults" entry.
