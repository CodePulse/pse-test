Resolution of the model parameters

All parameters in the model need to be given a value before the system of equations and variables for the flowsheet model is constructed. These parameters are the quantities in the model that do not vary during the simulation. Typical problems occurring during this stage are:

  • No value is given to a parameter
  • An array parameter is indexed out of its defined range
  • Two conflicting values are given to a parameter

No value given to a parameter

If no value is given to a parameter the error that is shown typically indicates which unit model, on the flowsheet, is missing a parameter specification. Check that the dialog for that unit model has no fields that are missing values.

Resolving parameters...
1 error(s) occurred while resolving parameters
Error in MODEL Gain_parameter_only_gML at line 3 (32):
Error P1001: Parameter Flowsheet.g1.gain(1) must be SET a value.

A parameter has not be specified for unit model "g1" on the flowsheet.

Value for the "Gain" parameter in unit model "g1" has not be provided

An array parameter is indexed out of its defined range

When a value is given to an array parameter outside of its defined range, an error of the type below, is displayed.

Resolving parameters...
1 error(s) occurred while resolving parameters
Error P1004: PARAMETER 'Flowsheet.s1.constant_value(2)' could not be resolved.

A parameter value for index 2 is given outside of the defined range for the parameter in unit model "s1"

For this type of problem, check that the specified range is consistent with the number of values given. For example, check that not more mass fractions are specified than there are components in the physical property package.
