Numerics

The Numerics tab is used to specify numerical performance and accuracy parameters.

Name Default Unit Description
Calculation mode Selector
Normal
All balances are used as normal.
Cold preset
All balances use simplified expressions. Used for initialisation and troubleshooting only.
Bed axial grid transformation Selector
None
Uniform grid.
Logarithmic
Non-uniform grid with more nodes towards the lower bound of the domain (inlet).
Exponential
Non-uniform grid with more nodes towards the upper bound of the domain (outlet).
User defined nodes
The grid is manually specified.
Pellet radial grid transformation Selector
None
Uniform grid.
Exponential
Non-uniform grid with more nodes towards the upper bound of the domain (pellet surface).
User defined nodes
The grid is manually specified.
Reaction rate mode Selector
Full
Reaction rates calculated as normal.
Constant
Reaction rates are given a constant value.
Post-calculation mode Selector
Full
Post-calculations are included in the calculation.
Constant
Post-calculations are excluded from the calculation. Used during the initialisation procedure.
Initialisation procedure Selector
Automatic (Fast)
Automatic (Fast) is a unit initialisation procedure that is optimised for speed. This initialisation procedure may occasionally fail for some problems. If this happens, use the Automatic (Robust) option.
Automatic (Robust)
Automatic (Robust) is a unit initialisation procedure that is optimised for robustness. This initialisation procedure should work in all cases but is generally slower than the Automatic (Fast) option.
Manual
Allows selection of any one of the existing initialisation procedures. This option is recommended for advanced users only.
Off
Do not use an initialisation procedure for this unit. This option can be used when the unit is initialised from a Saved Variable Set (SVS).
Reaction factor -
Reaction rate multiplier used during the initialisation procedure to activate the reactions
Number of elements in bed axial domain - Number of discretisation points used in the axial distribution domain of the model.
Number of elements in pellet radial domain - Number of discretisation points used in the pellet radial distribution domain of the model.
Discretisation order of pellet radial domain - Order of the discretisation method (Central Finite Difference Method - CFDM) used in the pellet radial domain . Allowed values are: 2, 4 or 6.
Bed axial domain transformation parameter - Parameter to be used in Logarithmic or Exponential transformation (see gPROMS documentation for further information).
Position of nodes in bed axial domain - Position of internal nodes for “User defined nodes” option
Pellet radial domain transformation parameter - Parameter to be used in Exponential transformation (seedocumentation for further information).
Position of nodes in pellet radial domain - Position of internal nodes for “User defined nodes” option
