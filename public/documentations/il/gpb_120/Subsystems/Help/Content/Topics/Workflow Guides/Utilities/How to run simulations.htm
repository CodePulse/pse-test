Executing simulations

Right-click on a Model entity and select 'Edit PROCESS' to create a new Process entity - with the same name – to simulate the Model. The “Run” icon in the menu will now appear green when you select either the model or the process. Clicking on this icon executes a simulation of the Process.

Executing a simulation

The process can be customised to include additional specifications, scheduled tasks such as sending results to MS Excel, or to customise solvers and other solution parameters for a specific problem. Multiple processes can be configured for the same model entity by copying and pasting an existing process and changing the text as required.

Hint
  • Always build flowsheets up sequentially. Execute a simulation after every change to ensure that the flowsheet has been specified correctly. It is much easier to debug a single unit than a whole flowsheet.
  • If the simulation fails, always check the structural analysis report first to make sure that the correct number of equations and assignments has been provided for the variables in the model.
  • Use annotations (images, text and value tables) to present important information quickly to users – it is a lot quicker to check flows on an annotated diagram than it is to browse through trajectories and model reports.
  • Use flowsheet layers to quickly toggle between different views of the flowsheet.
