Updating specifications and initial guesses with estimation results

A typical part of the workflow associated with parameter estimation activities involves taking the optimal parameter estimates that are generated by the calculation and using them to update the values of the parameters specified in the model specification dialog(s). This transfer can be done without any manual transcription of numerical values: in the section on Model Parameters of the comprehensive report, there exists a hyperlink as displayed in the highlighted area below which displays a dialog for exactly this purpose.

Estimated model parameters

The dialog allows the user to update values in specification dialogs and also intitial guesses for future estimation activities.

  1. update the values of the parameters in the model specification dialogs

  2. use these values as initial guesses for future estimation activities.

Using final estimates for updates

It is important to note that these two choices are independent of each other, providing advanced users with some additional flexibility. For example, it is possible to choose (1) but not (2). In such a case, a subsequent simulation or optimisation activity will indeed use the optimal parameter estimates. However, a new parameter estimation run (perhaps one involving fewer or more experimental data sets) will start using the default values of the model parameters.
