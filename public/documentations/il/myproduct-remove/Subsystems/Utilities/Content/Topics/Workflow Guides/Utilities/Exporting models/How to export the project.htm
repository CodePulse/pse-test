Exporting the project

Before exporting the project, some extra lines should be added to the process. The first is a ‘Monitor’ statement which is added at the top, just after the model declaration:

        UNIT
        
  Flowsheet AS model_entity_name

        
 

        

          Monitor
        

        
Flowsheet Site_costs_gML001.TotalCost ;

gPROMS language adding a Monitor statement

In this example, 'Site_costs_gML001' is the name given to the cost model when added to the flowsheet. This should be modified to provide the specified name. The purpose of this step is to provide a flag to the gPROMS remote execution engine (gO:Run) to utilise Model Pruning. This is a default option when running simulations and optimisation from within the software but is currently off by default in gO:Run. This step will not be required in the product release version.

Add a Schedule section at the end of the process, to create a Saved Variable Set, which is an output file listing the values of all variables in the model. This is very useful for viewing the results of the optimisation and allows the user to view more than just the values of controls, constraints and the objective.

        SCHEDULE
        
  Save "Results"

gPROMS language to create a Saved Variable Set

If the Process is simulated now, the trajectories section should be empty except for the 'TotalCost' variable, and a Saved Variable Set should be found in the results section. The file is now ready to be exported. Export the file by right-clicking on either the process or the optimisation (which must both have the same name) and choose ‘Export’. Keep the default setting of ‘Encrypted input file for simulation by gO:Run’ and press 'OK'.

Select the export directory, which is where the encrypted model will be placed. The input file name will by default take the same name as the process. Do not change this. The 'Encryption password' will also take the same name as the process. Edit this by adding “_*5tr0ng” at the end, and click on ‘Export’. Two new files (Settings.input, PhysProps.mfl) will be created in the export directory, and one sub-directory (input) will be created and should contain the encrypted process information (Processname.gENCRYPT) and, if the export was performed on the optimisation, the optimisation (ProcessName.gOPT) entity.
