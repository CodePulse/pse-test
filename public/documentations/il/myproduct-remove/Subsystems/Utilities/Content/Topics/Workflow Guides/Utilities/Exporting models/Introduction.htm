Exporting models for remote execution

This Workflow Guide describes the steps needed to convert a model developed in gML Utilities (such as described in the previous section) for running as an executable.

The main changes required are:

  • All control variables plus any other variable settings you would like to be able to specify for each run must be linked to a text file input. This usually includes settings that change over time such as contract prices, site demands, minimum and maximum flows, but may include any assigned variables.
  • Lines need be added to the process to tell the executable how to formulate the problem and to write a saved variable set of all the results following a simulation.

gML Utilities Beta currently does not provide an automatic process for this conversion and so instructions are provided below to apply these changes manually. The release version will include a wizard for preparing the gML Utilities project for remote execution and pre-populating the Excel template.
