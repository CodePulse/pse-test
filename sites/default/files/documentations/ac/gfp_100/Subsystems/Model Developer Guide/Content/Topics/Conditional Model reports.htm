Conditional Model reports

The appearance of all of the elements that can be part of a report can be made dependent of logical conditions. In the simple example below, a text and its colour depend on the test if a temperature has crossed a threshold.

<Conditional>
   <If expr="temperature_maximum .gt. Tlimit">
     <p STYLE="color: red">
       <B>WARNING: the temperature in the reactor exceeds 630K!</B>
     </p>
   </If>
   <Else>
     <p STYLE="color: green">
       The reactor operates at a safe temperature
     </p>
   </Else>
</Conditional>

Each conditional element or group of elements needs to be enclosed in a <Conditional> tag. Contained herein are the tags for the tags if/elif/else where if and elif can contain logical expressions as XML attributes. The expressions are standard gPROMS logical expressions, with the exception that the following comparison operators have to be used:

gPROMS language operator Conditional report operator
<  .lt.
<= .le.
>= .ge.
> .gt.

true and false are valid elements of expressions.

Two aspects are important to note:

  1. Any gPROMS variable or parameter that is within the scope of the report can appear in the expressions.

  2. Expressions are evaluated at the time defined by the report's time slider and the report will be completely redrawn upon a change of the time slider.

    Although gPROMS parameters are constant during the execution of a gPROMS Activity, values of parameters can be used as constant parts of expressions, for instance in order to determine static parts of the layout.
