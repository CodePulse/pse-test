How to select and define factors

Factors are user-specified inputs to a system which are uncertain or experience variability. Internally, a factor is mapped to a scalar element of an assigned gPROMS variable1If it is attempted to map a factor to a continuous distribution or an array, the selection dialog will denote this with an error mark and an explanation..

Factors are specified in the Factors tab and named according to gPROMS naming conventions, as long as they are unique within the GSA entity. All references to the factor within the GSA entity make use of this name rather than the variable path, for convenience. Factors can also be given a textual description and a set of tags.

The units of measurement shown in the Factors tab apply to all values associated with each factor throughout the GSA entity and its execution results.

For parametric and uncertainty studies, selecting the factors is straightforward. However, for sensitivity analysis it depends on the goal of the analysis:

  • To identify the most influential factors before an in-depth analysis, a large number of factors can be considered (see 'Elementary effects sensitivity method'). In this case, specify factors which are suspected to have an effect on the responses.
  • To determine which factors are most influential from a reasonably sized set, a smaller set should be used (see 'Uncertainty analysis' and 'Variance-based sensitivity method'). Available physical insight, knowledge of the system or reduction as described above, dictate the number and type of factors to consider.
