The set of physical properties supported by gPROMS

External physical property packages are interfaced to gPROMS via the latter's Physical Property Interface (PPI). The PPI recommends that a minimal set of physical property calculations are supported by any such external package. These are listed in the tables below and include most of the common thermophysical and transport property calculations used in the chemical industry.

Caution

Always consult the documentation provided with the physical property interface that you intend to use.

The physical properties listed in the tables below represent a minimal set of properties that physical property packages, interfaced to gPROMS, are recommended to support.

There is no guarantee that all (or any) of these properties are actually supported by a particular interface. Also, some interfaces may provide access to properties other than those listed in the tables below.

The units of measurement of the physical properties and the various quantities needed for their calculation may also vary from package to package.

There are a few points to note about the methods and their arguments listed in 'Constants and pure component properties' and 'Phase properties' :

  • Scalar methods (e.g. LiquidEnthalpy) return a single value, typically corresponding to a property of the mixture.

  • Vector methods (e.g. MolecularWeight) return an array of results, each element of which corresponds to a different component in the mixture.

  • Most methods receive three arguments, namely T, P and n.

    • T is a scalar quantity that corresponds to the temperature and should be in Kelvin.

    • P is a scalar quantity that corresponds to the pressure and should be in Pascal.

    • n is an array-valued expression that corresponds to the number of moles of the components in the mixture.

  • The enthalpy, entropy, internal energy and volume calculations return total quantities. However, the corresponding specific quantities can be obtained by normalising the mole numbers n,so that molar fractions are passed as arguments to the various methods.

  • The LiquidActivityCoefficients method that returns the values of the symmetric liquid activity coefficients can be called by the alias LiquidActivityCoefficientsSym. Note that in this case, the reference state for each component in the mixture is the pure component at the same T and P.

  • The following methods receive four arguments, namely the temperature (T), pressure (P), and two array-valued expressions that correspond to number of moles (n1, n2 or nL, nV). These are :

    • The SurfaceTension method where nL and nV are the number of moles of, respectively, the liquid and the vapour phases in contact with each other.

    • The methods that return the asymmetric activity coefficients (ActivityCoefficientsAsymConc, ActivityCoefficientsAsymMolal, ActivityCoefficientsAsymMolar) and the corresponding methods that calculate the natural logarithm of the asymmetric activity coefficients. In this case the first array n1 contains the number of moles of each component in the mixture, whereas the array n2 contains the number of moles of each component in the reference system.

  • For the vapour phase properties, the spelling VaporXX is also recognised as an alternative to VapourXX.

The methods presented in 'Equilibrium flashes' perform equilibrium flash calculations. The following points should be noted:

  • The number of moles n in the inputs list refers to the overall composition of the mixture.

  • The content and size of the results array, returned by the vector methods, will depend on the particular physical property package that is being used. It is highly recommended to refer to the appropriate documentation (see also: the Multiflash physical property interface for a particular example).

The methods presented in 'Mixture properties' internally perform T, P equilibrium flash calculations. The following points should be noted:

  • The number of moles n in the inputs list refers to the overall composition of the mixture.

  • The property methods, such as Enthalpy and Density, return the total properties of the system (and not for a specific phase, as is the case for the methods listed in the table on 'Phase properties').

    The total property values over all phases are calculated according to the following formulae.
    For Energy, Enthalpy, Entropy, GibbsFreeEnergy, HeatCapacity and Volume:

    Notation

      XT Total property
      Xmi Molar property in phase i
      N Total number of moles
      np Number of phases
      θi Molar fraction of phase i
    For Density:

    Notation

      ρT Total density
      Mw,i Molecular weight of component i
      V Total volume
      nc Number of components
      zi Total molar amount of component i
    For CompressibilityFactor:

    Notation

      P Pressure
      V Total volume
      N Total number of moles
      R Universal gas constant
      T Temperature
    For SpeedOfSound:

    Notation

      Total speed of sound
      Speed of sound in phase i
      ρT Total density
      ρi Density of phase i
      np Number of phases
      αi Volume fraction of phase i
