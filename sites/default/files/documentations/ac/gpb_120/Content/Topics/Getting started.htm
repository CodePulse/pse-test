Getting started with gPROMS ProcessBuilder

gPROMS ProcessBuilder is an advanced process simulation tool for model-based support of key process design and operating decisions. Built on the gPROMS Platform, gPROMS ProcessBuilder allows users to construct process flowsheets by dragging and dropping models, for basic and advanced unit operations, from a palette of model libraries. The process flowsheet can then be simulated or used in optimisation or model validation studies.

In addition to this guide, a review of 'Important information for gPROMS ProcessBuilder 1.2.0' is recommended. Reading 'Upgrading to version 1.2.0' may also be useful if issues are encountered whilst updating flowsheets from version 1.0.0 to version 1.2.0.

To get started directly with constructing and simulating flowsheet models, see Constructing flowsheets and Simulating flowsheets. The fundamental layout of the user interface and a detailed description of all its features are given in the gPROMS ProcessBuilder Guide.

The models for commonly used unit operations are included in the gPROMS Model Libraries (gMLs). With gPROMS ProcessBuilder open, the libraries can be loaded by accessing the File->Open/close libraries menu option or by clicking the Open Library button on the toolbar. The gMLs include the libraries shown in the table below. For help on configuring individual models or to see what the key equations for each model are, refer to the model reference documentation.

Library name Description License
gML Basics Sources/sinks, mixers/splitters, stream analysers and many other standard flowsheeting elements. gML:Multipack 1
gML Control Models to trade-off specifications between different unit operations and implement control schemes. gML:Multipack 1
gML Flow Transportation Valves, pipes, pumps and compressors. gML:Multipack 1
gML Heat Exchange Heaters, coolers, two-stream and multi-stream heat exchangers, evaporators and condensers. gML:Multipack 1
gML Mass Balance Mass balance-only models, including sources/sinks, mixers/splitters, conversion and yield reactors, component splitter and stream analyzer. gML:Mass balance
gML Reaction Conversion and Gibbs reactors, kinetic and equilibrium continuous stirred-tank (CSTR) and 1-D plug flow (PFR) reactors, with Arrhenius, Langmuir-Hinshelwood and user-specified reaction mechanisms. gML:Reaction
gML Separations - Adsorption Adsorption bed unit with axial and radial flows, with customisable templates for isotherms and mass and heat transfer coefficients, convenient definition of periodic adsorption cycles and schedules for the self-interacting bed approach. gML:Separations - Adsorption
gML Separations - Fluid-Fluid Flash drum, equilibrium tray and packed-bed distillation columns, absorbers with/without condenser, kettle reboiler, steady-state 1-D and 2-D rate-based absorption columns. gML:Separations - Fluid-Fluid
gML Signal PID controller, gain, delay, logic switches, transfer functions, dead zones, hysteresis, saturation, mathematical functions and basic operations, signal sources (ramps, steps, function generators), displays, plots, lookup tables, logic programing and other control elements for control design and data manipulation. gML:Signal

In addition to the unit operations contained in the gML libraries, it is possible to add custom unit operations using gPROMS' custom modeling capabilities. These are constructed by writing equations, describing the physics and chemistry of the process without the need for specifying how the model is to be solved. Models can include spatial or size distributions, and sophisticated condition handling is provided. It is possible to publish models in libraries for other users across the organisation.

To aid in flowsheet development and troubleshooting, a series of workflow guides are avaliable. The complete list of included workflow guides as well as a brief description of each one is given in the table below.

Workflow guide name Description
Constructing flowsheets Workflow guide describing how to add units to a flowsheet using the gPROMS Model Libraries, connect the units together and populating the units dialogs.
Simulating flowsheets Workflow guide describing how to run the simulation once the flowsheet is assembled, viewing and exporting the results.
Specifying process materials Workflow guide describing how to specify physical property packages to be used in a flowsheet. In addition, it also covers more complex cases where more than one physical property package is required.
Flowsheets with recycles Workflow guide describing how to handle the presence of material, energy and signal recycles in a flowsheet.
Trading off degrees of freedom Workflow guide describing how to easily trade off degrees of freedom, i.e. determine the value of one variable in order to achieve a desired value for another variable in the flowsheet.
Signals and control Workflow guide describing the use of the Signal library models for parallel calculations and how to interface these models with the unit operations, sources, sinks and stream analyzers.
Troubleshooting Workflow guide describing, in detail, how to troubleshoot a flowsheet that has failed to converge and how to get more information from the Execution output to aid in this process.
Custom modelling Workflow guide describing how to customise existing gML models, how to develop custom unit operation models and how to integrate these models into a flowsheet containing standard gML models.

A catalogue of example processes is included with gPROMS ProcessBuilder. These can be accessed via the Open an example project button on the toolbar. A description of each example can be found in the overview of the examples.

Apart from simulation, a flowsheet can also be used for purposes of model validation and optimisation. Model validation is sometimes required to adjust empirical parameters, such as reaction kinetic parameters or heat transfer coefficients, to fit observed process measurements. Rigorous mathematical optimisation can be used to substantially reduce operating and design costs or enhance profit by making design and operational decisions based on objectives that capture process economics and constraints.
