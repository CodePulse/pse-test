Important information for gPROMS ProcessBuilder 1.2.0

Physical property considerations

Reference state for enthalpy

All gPROMS ProcessBuilder models consider the reference state for the enthalpy to be defined on a chemical elements basis. Any physical property packages used need to be configured accordingly to avoid incorrect model predictions or convergence failures, especially for processes involving chemical reactions.

By default, Multiflash uses pure compounds as the reference state for the enthalpy datum. Therefore, this needs to be reconfigured to an elements basis. For information on how to do this, please refer to section Enthalpy definition of the Multiflash for Windows user guide included with gPROMS ProcessBuilder.

gSAFT also needs to be configured to use chemical elements as the enthalpy reference state.

Maximum number of phases

Unit operation models in gPROMS ProcessBuilder are currently limited to two phase (gas - liquid) processes. If the physical property package predicts more than two phases, the resulting predictions for unit operation models that involve phase separation may be incorrect. This affects the following unit operation models:

  • Separator
  • Column section
  • Distillation column
  • Distillation column refluxed
  • Distillation column reboiled
  • Evaporator kettle
  • Reactor stirred tank VLE
  • Stream analyzer (only when turning the Flash analysis option in the specification dialog to On)

Any other models will predict results correctly for three or more phases.

Initialisation for dynamic simulations

For steady-state problems and dynamic problems that are initially at steady-state gPROMS ProcessBuilder uses flowsheet initialisation procedures. These are active by default when a new flowsheet and process are created. For dynamic problems with dynamic initial conditions this flowsheet initialisation procedure needs to be deactivated. To deactivate flowsheet initialisation procedure functionality, change the SequentialInitialisationProcedure setting in the Execution category of the Solution Parameters for the corresponding process.

Units of measurement

In gPROMS ProcessBuilder1.2.0, there are certain situations where the units of measurement are the ones used internally for the solution of the underlying equations, and not the ones used to configure the unit operation models via the unit specification dialogs.

This is relevant in the following situations:

  • When viewing model reports the units of measurement may be different from the units used to specify the models.
  • When specifying model variables using models that employ a gMLBus connection, the specifications made into those models should be made in the internal units.
  • When using custom modelling to write equations or configure unit operation schedules that refer to variables in gML models, be aware that these variables are expressed in the internal units.
  • When specifying performance curves for compressors, pumps or expanders using a curve input file. The information specified in the input file needs to be expressed in the internal units.
Unit operation models

For a number of unit operation models there are known issues.

General
Flow reversal under dynamic operation.

In gPROMS ProcessBuilder1.2.0, the handling of flow reversal under dynamic operation is limited to the following models:

  • Material source
  • Material sink
  • Valve
  • Pressure changer
  • Junction
  • Stream analyzer
  • Adsorption bed

For these models, a corresponding model with the suffix "_reversible" is available in the model libraries. These are the models that should be used when handling flow reversal is a requirement.

gML Basics

Stream analyzer

The stream analyzer can be used to compute physical properties of material streams. The computation will typically result in a small increase in simulation time but should not affect results or models convergence.

gML Connectivity

gMLMaterial_to_PML_Material / PML_Material_to_gMLMaterial

These models can be used to convert streams between the gML library format used in gPROMS ProcessBuilder and the PML library format made available in earlier versions of gPROMS ModelBuilder. Results may be incorrect for (near-) pure component material flows at the boiling point.

gML Control

Balance and recovery models

These models are used to verify mass balances and predict recoveries. In gPROMS ProcessBuilder1.2.0, these models operate correctly only under steady-state conditions. They should not be used if the process involves unit operation models with mass or energy holdup under transient conditions.

gML Separation Fluid-Fluid

Separator

Dynamic simulations may not converge when separator vessels run empty or become completely full.
