Schedule

The Scheduling model can be used to defined a dynamic operating schedule. This can be done by connecting the units that are to be operated using the schedule to the Scheduling model using either gMLBus or gMLSignal connections.

To define an operating schedule, the Scheduling model can be added to an existing flowsheet. Then connections can be made between the schedule model and the units that are to be controlled according to the operating schedule. This operating schedule is configured in the Scheduling model by defining of a number of time periods. For each time period, a control input can be specified for each piece of equipment that is connected to the schedule model. The time periods are either fixed-length or can be terminated when a specified condition is met. The sequence of steps can be repeated a specified number of times. Each repetition is referred to as a cycle.

This model is primarily used to define operating schedules for periodically operated processes such as Pressure-Swing Adsorption (PSA) or Temperature-Swing Adsorption (TSA) processes.

The Scheduling model also permits the use of measured values at a certain time period as inputs at another time period. This allows the use of the Scheduling model to accelerate, for example, simulations of cyclic multi-bed processes by exploiting symmetry between the states of the bed.
