Executing multiple scenarios

Executing multiple scenarios is simply an extension of the single scenarios. Use the ’Add Period’ button in the Inputs sheet to add extra periods to run. This will create a new period input sheet, pre-populated with the same values as the last existing period, which can be modified as required. When the ‘Run All’ button is pressed, a steady state optimisation is executed for each period sequentially which can represent hourly, daily, weekly or any other time interval.

The results for each period are returned to the 'Results' sheet in sequential columns starting from column E.

Hint
  • Use the 'Flags' column to influence the initial conditions.
  • The character ‘r’ denotes a heading in column A and causes the code to ignore that row but treat subsequent rows using a ‘routing formulation.’ This is a linear reformulation of bilinear constraints which has been found to work well for routing decisions (steam valves and ejectors).
  • ‘X1’ & ‘X2’ where 'X' is any matching character can be used to customise the initial guess for on/off decisions. By default, all binary variables start set to ‘On’. Binary variables can be assigned pairs by entering "X1" and "X2" in Column B of each pair, where 'X' is a letter used only for that pair. This will modify the initial condition to always start with one of the pair on and the other off. This is useful in particular for two boilers feeding into the same header, where one is often on and the other kept for reserve. The preferred unit (for example, the more efficient of two boilers) should normally be assigned "X1" and the least efficient "X2".
  • ‘Max’ may be added to customise the Initial guess for continuous variables. By default all continuous variables start at their lower bounds ('Min'). If the 'Max' flag is added to a continuous variable, it will start instead from the upper bound. In this case (and in fact in general) it is important to ensure that the upper bound is realistic and not just a huge number.
