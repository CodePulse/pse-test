Controlling the automatic routing

By default, connections between Units on the topology are routed automatically in an orthogonal layout that avoids connections overlapping with Units and Flowsheet Annotations. The automatic routing is usually efficient enough and allows the continued editing of a topology while running in the background. For the incremental build-up or the modification of very large flowsheets, routing may still take more time or resources than desired and it is possible to interrupt it while it is running: the grey square next to the routing menu turns red after some time, , and pressing it will cancel the routing. By choosing Route all connections, , from the routing menu, it is then possible to route all connections in their entirety.

It is also possible to turn the automatic routing off entirely for a certain period and to turn it back on again. This can be achieved using the routing drop-down menu of the Topology editor tool bar. This flag is specific for a flowsheet and does not affect any other from the same Project. The setting is stored persistently upon saving a Project.

Automatic routing enabled or disabled

If automatic routing is turned off, it is possible that skewed, non-orthogonal connections are generated, for instance by moving Units or when creating new connections. In such a case, it is possible to fix only selected skewed connection, , or all of them .

In all modes it is possible to route only selected connections by using the Route selected connections command, , performs the same orthogonal routing algorithm, but only on the selected connections.

Further options for controlling the routing behaviour are available in the user preferences.
