Creating a custom databank for SAFT-g Mie

In the case of a databank that corresponds to the SAFT-g Mie EoS, the <gsaft> element is required to have a single <compounds> element as well as a single <groups> element as children element. The <compounds> element is only allowed to contain a single or multiple <compound> elements, with each element containing the parameters that are required to describe a compound. The <groups> element can in turn contain multiple <group> elements, where each element used to specify the required parameters that describe a functional group. Following the <groups> element, the interactions between different functional groups can be specified within an optional <crossInteractions> element, which contains a <crossInteraction> element for each specified interaction.

Note that each <crossInteraction> element is always in reference to declared groups in the <groups> element, and the specification of the groups to which a <crossInteraction> element refers to is done in terms of the group1 and group2 attributes. A small example that demonstrates how a databank that specifies the compounds n_butane, 1-pentanol, and 1-hexylamine, together with the required functional groups (in this case assumed to be the CH3, CH2, OH, and NH2 groups) elements as well as some of the cross interactions between these is shown below. Note that the contents of the <compound>, <group>, and <crossInteraction> elements are omitted for brevity; these are discussed in more detail in the following sections. 

<gsaft theory="gc_mie">
  <compounds>
    <compound name="n_butane">
      ...
    </compound>
    <compound name="1-pentanol">
      ...
    </compound>
    <compound name="1-hexylamine">
      ...
    </compound>
  </compounds>
  <groups>
    <group name="CH3">
      ...
    </group>
    <group name="CH2">
      ...
    </group>
    <group name="OH">
      ...
    </group>
    <group name="NH2">

  </crossInteractions>						
</gsaft>
Compound definition

In a databank that is to be used with the SAFT-g Mie EoS each compound is specified within a <compound> element. A valid <compound> element requires the specification of the name of the compound by setting the name attribute to a valid string, whereby each compound must have a unique name. Since the SAFT-g Mie EoS is a group contribution approach, compounds are defined in terms of the types and multiplicities of the functional groups that they comprise. The specification of the functional groups that a compound comprises is done by adding a <groupMultiplicities> element, within which a <groupMultiplicity> element for each type of functional group is declared; in the <groupMultiplicity> element, the group type is declared as the value of ref attribute of the element and is required to be a valid string, whereas the group multiplicity is defined as the value of the <groupMultiplicity> element and is required to be a integer. For the compounds n_butane, 1-pentanol, and 1-hexylamine used in the example above the group multiplicities are as show in the table below.

The <dielectric> element

This element gives the coefficients of a linear correlation for the value of the dielectric constant as a function of temperature. The parameters v and t are specified as the values of the corresponding attributes of the <dielectric> element, as shown below for the case of water as an example (note that water is assumed to comprise a single occurence of the H2O group).

<gsaft theory = "gc_mie">
  <compounds>								
    <compound name = "water" >
      <groupMultiplicities>
        <groupMultiplicity  ref = "H2O">1</groupMultiplicity>
      </groupMultiplicities>
      <dielectric v = "0.3777e-3" t = "1403"/>
    </compound>							
  </compounds>
  <groups>						
    ...
  </groups>
</gsaft>

Group definition

In a databank that is to be used with the SAFT-g Mie EoS each group type is specified by adding a <group> element within the <groups> element in the databank. A valid <group> element requires the specification of the name of the group by setting the name attribute to a valid string, whereby each compound must have a unique name. Each <group> element can contain any of the elements presented in the table below, where each element can be of single occurrence. Note that these elements can be of two types, as listed in the Element Type column. Simple elements are elements simply contain a value (e.g., <mw>18.015</mw>) and have no attributes and/or child elements. Any other element is listed as Complex; the exact structure of these elements will be discussed in more detail.

The <heatCapacity> element

This element contains the values of the contribution of each group type to the ideal gas heat capacity, which is calculated for each compound based on the Joback and Reid group contribution approach. The values of four coefficients need to be provided (a, b, c, d) as separate elements within the <heatCapacity> element, as shown below for the case of the CH3 and CH2 groups.

<gsaft theory = "gc_mie">
  <compounds>
    ...

    <group name = "CH3">
      ...
      <heatCapacity>
        <a>19.5</a>

        <c>0.000153</c>
        <d>-0.0000000967</d>
      </heatCapacity>
      ...
    </group>

      ...

        <a>-0.909</a>
        <b>0.095</b>
        <c>-0.0000544</c>
        <d>0.0000000119</d>
      </heatCapacity>
      ...

  </groups>
</gsaft>

The <associationSites> element

This element is optional and is used in the case of associating (hydrogen-bonding) groups for defining the different types of association sites on a group, as well as the multiplicity of each association site type. The element can contain multiple <siteMultiplicity> elements, with each element used for the specification of each site type. The site type is specified as the value of the name attribute of each <siteMultiplicity> element and is required to be a valid string. The multiplicity of each association site type is then specified by setting the value of the <siteMultiplicity> element accordingly, which is required to be a positive integer. The <selfInteraction> element

The <selfInteraction> element contains the parameters that describe the self interactions between a given group. These are grouped in two different elements, with the <dispersion> element containing the parameters that describe the dispersion interactions, and the <association> element, where the parameters that describe the association interactions are placed. Note that the <dispersion> element is required within a <selfInteraction> element, whereas the <association> element is optional and can only be declared for associating groups.

The <dispersion> element contains the parameters of the Mie interaction potential for the self-interaction of the group. Together with the <numberOfSegments> and <shapeFactor> elements, the <dispersion> element provides the set of parameters that are required by SAFT-g Mie. The parameters that are specified within the <dispersion> element are listed in the table below.

The <association> element is used for the specification of the parameters that describe the association interaction (modelled by the square-well potential) between site types of a group. Each site-site interaction is described by an <interaction> element, whereby the site types to which the interaction refers are declared as the values of the site1 and site2 attributes of the <interaction> element. Note that the specified site types must have been previously declared in the <associationSites> element of the group, as discussed above.

  • In the example presented above the <interaction> element describes the association interaction between sites of different type. Association interactions between sites of the same type can also be declared in the same fashion, simply by assigning the same value to both the site1 and site2 attributes of the <interaction> element.
  • Site types can be assigned to the site1 and site2 attributes of the <interaction> element in any order; the association interactions for site types of the same group are symmetric, i.e., either of the elements <interaction site1="e" site2="H"> and <interaction site1="H" site2="e"> can be used to specify the same interaction.
Cross-interaction specification

The specification of cross interactions between groups is optional, and each <crossInteraction> element is used to specify the interaction between two groups. The groups to which each <crossInteraction> element refers to are specified by assigning the corresponding compound names to the group1 and group2 attributes of the <crossInteraction> element. Note that:

  • The values (i.e. group names) assigned to the group1 and group2 attributes must be valid strings and must correspond to names of groups previously declared in the databank within the <groups> element
  • A cross interaction between two groups can be specified by assigning the corresponding group names to the group1 and group2 attributes in any order. However, each cross interaction must have a unique definition, i.e., only one <crossInteraction> element that refers to two specific groups is allowed in the databank.

The structure of a <crossInteraction> element follows closely the case of the <selfInteraction> element.

The specification of the cross dispersion interactions is done within a <dispersion> element. As for the case of self interactions, the <dispersion> element can contain single <epsilon>, <sigma>, <lambdaAttractive>, or <lambdaRepulsive> elements. The specification of the parameters is optional, and if they are not specified the corresponding parameter values are calculated by means of the appropriate combining rules. 2 For more details on the combining rules employed see: V. Papaioannou, T. Lafitte, C. Avendaño, C. S. Adjiman, G. Jackson, E. A. Müller, and A. Galindo, "Group contribution methodology based on the statistical associating fluid theory for heteronuclear molecules formed from Mie segments", The Journal of chemical physics, 140(5), 054107, (2014).

The specification of cross association interactions is done within the <association> element, where a single <interaction> element is needed for the interaction between two site types. For the specification of the association interaction between site types on different compounds the order in which the site types are declared in the site1 and site2 attributes of the <interaction> element needs to follow the specification of the groups (group1 and group2 attributes) in the <crossInteraction> element. Namely, the site type specified in the site1 attribute, must be a site type declared within the group referenced in the group1 attribute of the <crossInteraction> element. The same holds for the site2 and group2 attributes. Note that within each <interaction> element the specification of both parameters (epsilonAssoc, and bondingVolume) within the corresponding elements is required; no combining rules apply in this case and any parameter not declared is assigned the value 0.
