gOPCFPI event synchronisation

gOPCFPI can synchronise with events on the OPC server when tag values become larger or smaller than, or equal to a specified value. It is also possible to wait until a specified duration in real time has elapsed. The syntax for these synchronisations is as follows:

PAUSE SIGNALID “ZeroTime: wait <wait duration>“
Continue after the real time duration of <wait duration>, where the duration is measured in 'modelTimeUoM'
PAUSE SIGNALID “ZeroTime: when <fpi tag> > <expected value>“
Continue as soon as the specified client tag is larger than the expected value.
PAUSE SIGNALID “ZeroTime: when <fpi tag> = <expected value>"
Continue as soon as the specified client tag is larger equal to the expected value.
PAUSE SIGNALID “ZeroTime: when <fpi tag> > <expected value>"
Continue as soon as the specified client tag is smaller than the expected value.

The gPROMS execution is resumed immediately once the condition is satisfied. No further real-time synchronisation is performed.
