Experiments and measurements

To define the experiments that will be used in the Parameter Estimation problem, click on the Experiments and measurements tab and then click the “Add…” button at the top right of the window.  This will open a new window that includes a project tree with all the experiment entities that have been defined in this project or a cross-referenced project.  In the Add experiments dialog, check the boxes of the experiments that you wish to include and select OK.  After all the experiments have been added you may want to consider the order of the experiments in the table which can be adjusted using the Move up and Move down buttons.  The order of the experiments in the table corresponds to the order in which they will be processed during the parameter estimation calculation.

Once the experiments are selected, a list of the measured variables associated with those experiments is formed in the “measurements” window of the “experiments and measurements” tab.  See above for an example. Experimental measurements are taken using sensors: the uncertainty of the measurement is a property of the measurement technique associated with the sensor. When solving a Model Validation problem all measured Variables are associated with a sensor.  Some of the variables can also be grouped under the same sensor.  In order to do this, select the measured variables from the “measurements” list and group them to form sensor groups.  Each sensor group can have one or more measured variables.  Each sensor group is associated with a variance model.  The variance model of a given sensor comprises information associated with the variance of the error of the measurement produced by the sensor.  The errors of the measurements are assumed to be statistically independent and normally distributed with zero mean.  There are several types of sensor variance models.  These can be considered to take the general form:

It is possible to group measured variables from different experiments that have the same variance model by highlighting them in the measurements table and clicking “Group selected”.  A new window will appear prompting you to select a variance model.  Once all measurements have been assigned a sensor model the tab should look similar to that below.

Best practice> Measurement weighting due to variance. The parameter estimation objective function (maximum likelihood) attributes weight according to the variance of a measurement. For measurements that may take very small values or span a large range, it is recommended to use the linear variance model rather than the constant relative variance model. This is because this model will ensure that very low measurements are not given an unnecessary high weight, thus biasing the results of the parameter estimation.
