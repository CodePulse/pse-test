How to select responses

Responses are the user-specified outputs from a system, whose values are functions of the factors and are computed via simulation of the model or flowsheet. Internally, a response is mapped onto a scalar element of an unassigned gPROMS variable.

Similar to factors, responses can be specified in the Responses tab and named according to gPROMS naming conventions, as long as they are unique within the GSA entity. All references to the response within the GSA entity make use of this name rather than the variable path, for convenience. Responses can also be given a textual description and a set of tags.

The units of measurement shown in the Responses tab apply to all values associated with each response throughout the GSA entity and its execution results.

It is possible to disable responses via a checkbox, which is enabled by default. Disabled responses are ignored during the execution of a GSA activity.

Response values are recorded at the simulation horizon endpoint. For steady-state simulations, this occurs when time is equal to zero, whereas for dynamic models, it occurs at the final point of the simulation. In future versions, it will be possible to specify time grids where the values of responses are reported (however, a temporary workaround is available).

Responses tab

Selecting responses is typically more straightforward than selecting factors and is dependent on the purpose of the analysis:

  • System understanding: KPIs or CQAs
  • Model understanding: Typical variables of interest, for example, temperature, pressure, composition, flow rate, among others
  • Selecting parameters to estimate: Measurements used in parameter estimation
  • Effect of uncertainty in parameters: KPIs such as yield, cost and safety constraints, among others
  • Selection of decision variables: Objective function and constraints
  • Effect of uncertainty in decision variables: KPIs such as yield, cost, safety constraints and objective function, among others
  • Effect of model uncertainty: Entirely case dependent.
