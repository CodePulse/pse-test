Case 3: Parameter uncertainty effect on KPIs

Parameter values used in models are usually sourced from literature, expert knowledge, historical data and/or parameter estimation using experimental data. Therefore, uncertainties are inevitable. GSA can be used to assess the effect of parameter uncertainty on KPIs and determine whether they are relevant. If the effect cannot be ignored, a more accurate parameter estimation may be required.

Create a GSA entity named 'Case3_ParaUncertainty'. The settings for the General tab are shown in the figure below. In this example, uncertainty analysis is performed with the quasi-random Sobol sample generation method.

Case 3: General tab

Specify the parameters of interest as factors in the Factors tab. Either use the 'Select...' button or enter the full variable path into the 'Variable in model' column, pressing Control+Space to use autocompletion.

Select each factor and click on the 'Edit...' button to specify all other factor settings. Note that, since only variables are eligible as factors, all parameters of interest must be moved from the PARAMETER section to the VARIABLE section of the model before the GSA entity is set up.

In this example, since reaction 1 is the dominant reaction, according to the results of Case 2, a parameter estimation can be performed to estimate the activation energy and pre-exponential coefficient of reaction 1 (refer to parameter estimation entity 'Par_est_1' in the GSA example Project for details).

According to the parameter estimation results, both parameters follow a normal distribution. Use this information to specify the 'Probability distribution' column for reaction 1.

Currently the covariance matrices of correlated factors do not return the correct factor space ellipsoid. This is a known issue which will be resolved in a future release. Therefore, assume that there is no covariance, only variance. According to the results of Case 2, reaction 2 is not relevant and will be ignored.

Specify the responses in the Responses tab. Similar to factors, either use the 'Select...' button or enter the full variable path directly into the 'Variable in model' column, pressing Control+Space to use autocompletion.

Case 3: Responsestab

The 'GSA language' tab is automatically updated once tabs General, Factors and Responses are set up. Launch the execution by right-clicking on the GSA entity and selecting 'Analyse system' or pressing icon in the toolbar. 

Case 3: Executing GSA

The 'Analyse system' dialog will be displayed. Change the Case name and select the desired number of workers for parallel computation. Note that this number should not exceed the total number of logical cores on the machine.

Once execution completes successfully, right-click on the 'System Analysis Group' folder and select 'New result element...' to create result elements. Alternatively, copy and paste result elements from other Cases. Note that the 'Factor Sensitivity Table' is not applicable in the context of uncertainty analysis and will not be populated with results.
