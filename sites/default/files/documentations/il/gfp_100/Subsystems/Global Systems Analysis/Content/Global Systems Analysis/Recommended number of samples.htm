Recommended number of factors and samples

The different types of number of samples generated in GSA are distinguished as follows:

  • The number of discrete deterministic samples is calculated directly from the specification of deterministic factors.
  • The number of uncertainty scenarios/probability distribution samples is user specified.
  • The number of total realisations of base activity to be calculated depends on the type and method of analysis and is determined as follows:

    • Uncertainty 

      Notation

       NdNumber of deterministic cases
       NNumber of uncertainty scenarios (user defined)

    • Elementary effects sensitivity

      Notation

       NdNumber of deterministic cases
       NNumber of uncertainty scenarios (user defined)
       KNumber of probabilistic factors

    • Variance-based sensitivity

      Notation

       NdNumber of deterministic cases
       NNumber of uncertainty scenarios (user defined)
       KNumber of probabilistic factors

The number of uncertainty scenarios/probability distribution samples should be specified for each type and method of analysis according to the guidelines below:

  • Parametric sensitivity studies: The number of samples is based on the number of enumerated and gridded space definitions.
  • Uncertainty: The number of samples is typically large in order to return a reasonably accurate estimate of the uncertainty in the responses, in the order of hundreds to thousands.
  • Sensitivity: The number of specified samples depends on the type of sensitivity analysis. Note that the number of samples required is (much) larger than the number of specified samples (due to the fact the indices must be calculated for each sample point, which requires multiple points for each sample point).
    • Elementary effect: A small number (20-100) is typically required, in the order of hundreds in rare cases.
    • Variance based: A very large number (>1000) is typically required.

While almost any model can be used for parametric, uncertainty and elementary effects sensitivity analysis, variance-based analysis is more computationally expensive, so computationally demanding models and/or models with a large number of factors will either take too much CPU time for practical use (even if many parallel workers are used) or not produce reliable results if too few points are used.

It is recommended that the number of factors be kept to a reasonable minimum. For example, specifying beyond 20 factors for all but elementary effect analysis will require a large number of evaluations and excessive CPU time, even on a machine with many cores. The total CPU time is dependent on a number of elements, for example, size and complexity of model, number and range of factors, method used and hardware configuration, among others. Therefore, starting small and increasing the complexity of the analysis as needed, is recommended.

To confirm an appropriate number of samples are being used, check for convergence by repeating the analysis for a larger number of samples and verifying that the results have not changed significantly. Mean values, quartiles and/or total effect indices are some examples of results which can be compared, to confirm convergence.
