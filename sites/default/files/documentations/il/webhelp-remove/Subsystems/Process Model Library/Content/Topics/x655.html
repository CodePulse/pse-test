Specification options

With the exception of trivial models (e.g. Dead End), all PML models have a Specification dialog that conforms to the following basic requirements:

  • All relevant Parameters and Selectors in the model are compulsory specifications.

  • Variables for ASSIGNment or as INITIAL conditions are optional specifications.

In order to allow for efficient specification of the most commonly used sets of specifications there are standard and advanced configuration modes. As an example of this functionality, the figure below shows the standard and advanced modes of specification for the Pipe PML Model. The advanced mode is typically used to trade specifications (e.g. between different units) and it is recommended that this mode only be used by users that are familiar with the PML models.

The Standard mode is characterised by the following properties:

  • There may be multiple sub-groups to support different specifications. For example, in the PID controller model there are sub-groups for controller class (P, PI, PID, PD, I, D) and the controller mode (automatic, manual & cascade).

  • Irrespective of which sub-groups are selected the number of Variables specified by the Specification dialog for ASSIGNment are equal to the number of Variables minus the Equations for that Model (assuming that the necessary stream and control port connectivity is in place).

  • All Variables specifications are of type "Optional On". This means that you still have the opportunity to turn off a variable specification if an equivalent specification is made elsewhere in the flowsheet.

  • For dynamic models there are always at least two options for initial conditions – steady-state and dynamic. Multiple dynamic options are allowed, and these are given appropriate names. For example, the Flash_drum model has the following options for initial conditions:

    • a steady-state option

    • a dynamic option where liquid level, Pressure and all but one of the molar fractions of the mixture are specified

    • a dynamic option where Temperature, Pressure and all but one of the molar fractions of the mixture are specified

The Advanced specification mode is characterised by the following:

  • All Variables that can be accessed from the Standard mode are also accessible from the Advanced specification mode. In addition to these there are less commonly used variable specifications and advanced parameters whose values can be changed.

  • The Variables shown as the default specifications in the Standard specification mode appear as the default specification choices in the Advanced mode, but these may be overridden.

Complex models may have multiple Specification dialog tabs to facilitate ease of use, particularly if there are too many specifications to conveniently fit onto one tab. In such cases the information is divided into logical groups and the different tabs are given appropriate names. An example is a reactor model where the kinetic reaction data is found on its own tab, with the remaining data found on a Configuration tab.

Dynamic models always have an additional "Initial conditions" tab where all the specifications that determine the initial conditions of the model appear. This tab may include SELECTORs and PARAMETERs as well as VARIABLEs, if these are appropriate for the different combinations of allowable initial conditions.
