Workspaces

Workspaces allow users to save and restore complete work environments consisting of libraries, projects and result cases. Examples for such work environments are

  • the currently open set of projects or
  • a list of the libraries and projects belonging to a specific work project that is being undertaken.

In workspaces, gPROMS ProcessBuilder stores lists of libraries, projects and result cases and allows opening them as one. There is always one workspace active even if the user has not specifically defined one – the default workspace. This workspace is saved on exiting gPROMS ProcessBuilder and re-loaded upon start-up, unless the preferences for workspaces have been specified otherwise: they allow to define for the start-up if

  • always the last used workspace is opened
  • a clean environment is chosen or
  • a choice of known workspaces should be presented.

Workspace preferences

Workspaces can easily be created by using Switch workspace… from the File menu. They can be created as empty ones or as populated by the currently open projects. Workspaces carry a name and are saved as files with the extension .gWS; any location on the file system can be selected, although good practice suggests keeping them in one location. Another option is to keep together libraries, workspaces and related projects below one common location (e.g. for distributing all files belonging to a certain work project). Such a distribution is supported by workspaces because all paths to the files are stored as relative ones.

Workspace creation

Workspaces are saved when closing gPROMS ProcessBuilder or when switching workspaces. Furthermore, the Save all menu command includes workspaces in the save process. A confirmation dialog appears that prompts for checking each individual item. Saving changed workspaces

Switching of workspaces brings up a list of known workspaces in a selection dialog (after a prompt to save the presently open workspace, if necessary). If none have been defined yet, this list consists of the default workspace and a clean default workspace only. Should the desired workspace not appear in the drop-down list, it is possible to browse the file system. Note that switching to a new workspace replaces the entire set of currently open projects with the ones of the new workspace. This also happens when workspaces are opened by dragging-and-dropping a workspace file onto the GUI.

Switching workspaces

 

The option Clean default workspace loads the factory default workspace and makes it the new default.

Launching gPROMS ProcessBuilder with a workspace as an argument (e.g. by double-clicking on a workspace file in Windows explorer) also loads all files of the workspace.

The currently active workspace is displayed in the title bar of gPROMS ProcessBuilder, unless it is the default workspace.
