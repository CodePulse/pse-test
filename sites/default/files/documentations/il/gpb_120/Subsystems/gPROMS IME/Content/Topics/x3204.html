User preferences

Many of gPROMS ProcessBuilder's features can be customised by the user in a preferences dialog that is opened from the Edit menu. There are several preference categories, the main options for each are highlighted in the table below.

User preferences dialog

User preferences
Categories Available configurations and specifications
Workspaces and projects
  • Which action should be taken regarding the workspace at start-up:
    • open the last used
    • display a completely clean environment without any libraries or projects loaded
    • allow choosing a workspace.

  • The number of recently opened projects to appear at the bottom of the Project menu.

  • Whether to make backup projects when saving or not.

  • Store creation and modification timestamps.
  • Display empty groups.
View

  • The look and feel of gPROMS ProcessBuilder. Two options are available: Windows1On the Windows platform, this is the default for fresh installations. It is recommended because it offers a better integration with the operating system, for instance the file dialogs. and Metal.

  • User-interface scaling. This defaults to the system setting and is useful only for HiDPI screens.

  • Individual gPROMS ProcessBuilder messages may have been set not to be shown again, but this can be reset.
Editors
  • Auto popup of language assistance; also called 'tooltips'.
  • The initial view tab of Processes and Tasks: the graphical Schedule view, gPROMS Language or properties. For Processes, the Solution Parameters are an additional option.

  • General text editor settings, such as tab size, the display of the status bar, line numbers, text limit width or a text limit line.

  • Actions of the middle-mouse button.
Fonts and colours

  • Font properties for family and size.

  • Syntax highlighting attributes.
Flowsheeting

  • Layout options of the topology view:

    • Colours and font sizes
    • Zoom to fit on resize
    • Never route when opening topology: this is only useful when large flowsheets are normally opened that have to be re-routed upon opening, for instance due to changed source libraries.
    • Route connections around obstacles on hidden layers: this can be of particular interest when certain layers are turned off in the topology view but are visible in printouts.

  • Flowsheet validation options:

    • Report units using default options: this will cause it an error if a Unit has any specification set to its defaults
    • Validate hidden attributes
    • Show model warnings on topology: warnings will not only be shown in the Project Tree, but also on the topology; if deselected, only errors will be shown
    • Show coloured borders around units with errors and warnings: this option can be used to highlight errors and warnings stronger than normal.

  • Layout options of unit specification dialogs:

    • Show model version in titles of specification dialogs
    • Formatting for distinguishing attributes with default from specified values.
Activity execution

  • Whether projects are saved prior to an activity execution.

  • The priority of executed activities.

  • The look and feel of the execution output window.
Activity defaults

  • Defaults of the the Case configuration and execution control dialog.
Results

  • Whether or not to show data points on graphs.

  • Whether graphs are shown in low-resolution mode.

  • The default number format.
Comparison

  • The colours used for text and group comparison.
File locations

  • The location of the temporary directory used by gPROMS ProcessBuilder. This is the execution directory used to run model based activities.

  • The default starting location for the file dialog used for opening and saving projects.

  • The search path for any software components used by gPROMS, e.g. Foreign Objects, Foreign Processes etc.
Schedule

  • The default width of schedule elements.
Licensed capabilities
  • Choice of optional, licensed capabilities gPROMS ProcessBuilder should start with. If the chosen capabilities are enabled upon a restart, depends on the availability of their corresponding licenses. Should no licenses be available, for instance because all available ones are in use, gPROMS ProcessBuilder will notify the user about this and allow to disable the capability request.
Advanced

  • Whether to display the interface language tab which offers direct access to XML code for the design of specification dialogs.

  • Above which size are text files treated in binary form to save space.

  • Configuration options related to backward compatibility of PSE's licensing.

    Warning

    Do not change any of the License management defaults unless instructed to do so by PSE support.
