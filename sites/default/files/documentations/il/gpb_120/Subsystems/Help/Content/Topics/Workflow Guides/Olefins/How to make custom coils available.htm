Making custom coil models available in the Furnace dialog

Copy the coil_interface_custom_gML model to the working project

Once the custom coil is created (see 'Creating custom coil models'), it can be selected from the Furnace model dialog. This can be done by first copying the coil_interface_custom_gML model to the working project. This model is in the gML Olefins Templates - Custom Modelling project, which can be accessed by clicking on the Open an example project icon, and navigating to the Templates directory. This model must not be renamed.

Edit the coil_interface_custom_gML model

The coil_interface_custom_gML model allows different custom coil models to be available in the Furnace dialog. Thus, different custom coil models can be selected in different furnaces in the flowsheet. The gPROMS language of the coil_interface_custom_gML model needs to be edited so that it accounts for the new coil model.

The new custom coil needs to be added to the UNIT section of the gPROMS language. This is illustrated in the image below.

In this case, my_custom_coil_1 is an instance of the custom coil model my_custom_coil and has the size of parameter use_coil_custom("my custom coil"). For more details on how to construct custom coils see 'Creating custom coil models'.

The next step consists in editing the SET section to include the newly created custom coil in the list of custom coils and adding connectivity equations for the length. This can be done using the template code provided (as commented code) and editing the relevant parts (e.g. the instance name should be changed from COIL_CUSTOM to the name given to the instance of the custom coil added, which in this example is my_custom_coil_1). The name given to the coil in the list of custom coils is the name that appears for that coil in the dialog of the Furnace model. For this case it will be my custom coil. Once this is completed, it is necessary to edit the TOPOLOGY section, so that the ports are appropriately connected.
