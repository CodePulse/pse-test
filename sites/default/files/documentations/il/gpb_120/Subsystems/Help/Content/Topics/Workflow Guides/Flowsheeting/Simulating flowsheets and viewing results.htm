Simulating flowsheets

Creating a process

To execute the simulation of a flowsheet model in gPROMS ProcessBuilder, a Process associated with the flowsheet model needs to be created. The process contains all information required for a particular simulation of a flowsheet.

The most convenient way to create a process, for an existing flowsheet model, is to right-click on the flowsheet model. A context menu will appear.

Context menu obtained after right-clicking on the flowsheet model in the project tree.

Next, select “Edit process” from the context menu. A new process will be created. It will appear in the “Processes” tab of the project.

A new process has been added to the project with the same name as the flowsheet model.

Example

Create a new project and within the project create a new flowsheet model entity. Create a flowsheet model (See 'Constructing flowsheets').

New create a new process for that model.

Executing a simulation

After a process has been created for a flowsheet model, this Process can be executed. To execute the simulation a flowsheet model, select the associated Process and then either click Simulate in the toolbar, select Activities -> Simulate selected model from the menu or press F5. A 'Simulation settings' dialog pops up, in which changes to simulation settings can be made. In most cases its sufficient to just press 'OK', after which the simulation is started.

Hint

Press SHIFT+F5 instead of F5 to run the 'Process' directly without the 'Simulation settings' dialog appearing. This is particularly useful when running a simulation for the same process multiple times.

When a simulation run is started, the 'Execution output' window is opened. If the simulation is successful the flowsheet topology will be shown on this window. This flowsheet topology is identical to that shown for the flowsheet model in project, where the flowsheet is constructed, although it cannot be modified. Instead, the execution output it can be used to view and obtain results for the simulation. On the topology, certain unit models may also appear with a red hue. This indicates that a warning has been flagged up for that unit during a simulation. See'Troubleshooting' for an overview on how to interpret warnings.

Flowsheet topology shown in the execution output window

There is also an 'Output' tab on the execution output window. This tab may be shown by default if the simulation is not successful, but it can also be accessed by clicking on the 'Output' tab at the bottom of the execution output window. On this tab, messages are shown from the gPROMS simulation engine in text format. The information displayed in this window can be used to see how the simulation is progressing. Furthermore, when an error occurs during the simulation, the information in this window will indicate the cause of the error and the events leading up to it.

Example of the execution output for a successful simulation.

When a simulation has failed for a particular reason, a dialog will pop up. To troubleshoot failing simulations, see 'Troubleshooting'.

Simulation failure dialog

At the time when a simulation is started, a case file is also created in the project tree. A case file is a copy of the project at the time when the simulation was run, together with the results of the simulation. It can be saved to disk like a normal project file. The folders in the case file can also be used to access the simulation results.

Case file

Example

Click on the 'Open an example project' toolbar button and select from the "Flowsheeting" directory the "Flash separation.gPJ-PB" example project file. The project will open and appears in the project tree. Open the 'Processes' folder in the project, and select the process "Flash_separation_dynamic_initial_conditions". Now press F5. The 'Simulation settings' dialog appears. Press 'OK' to start the simulation.

Execution output window for 'Flash separation' example

Hint

The 'Simulation settings' dialog that appears after pressing F5 has a number of options for how the simulation is executed. The options used most often are:

  • Delete previous Cases from the same Activity Entity

    Useful when a simulation of the same process is repeated a number of times (often to troubleshoot issues or investigate the effects of changes in inputs). It will prevent a series of case files being created. Instead a single case file be updated with the results of each subsequent simulation.

  • Reporting interval

    For dynamic simulations, this permits the adjustment of the reporting interval, prior to the simulation. A Reportinginterval that is too large will mean that important features, in the trajectories for the dynamic simulations, will be missed. A Reporting interval that is too small may result in the generation of an excessive amount of results data.

  • Release license after execution

    Can be unchecked to retain the license after execution. Can be useful for troubleshooting failing simulations (see 'Troubleshooting')

Analysing the results

When a simulation has been successfully executed, the results of a simulation can be inspected in a number of ways.

Flowsheet topology

The Topology tab on the execution output window shows the flowsheet topology. This topology cannot be modified but it can be used to navigate to results of individual unit models.

Unit model reports

Double-clicking on individual unit operation models, sources, sinks and stream analysers brings up the results for these units in a separate window.

Results for the separator "Drum"

Some units have multiple tabs for each report.

Tab on report Type of information
Main Key information for the unit operation.
Operation Information related to how the unit operation is operated.
Costing Information related to the capital and operating cost for the unit operation. For Sources and Sinks it shows the material cost.
Dynamics When a dynamic simulation is performed, this tab will show dynamic
Warnings When a warning is generated the unit icon changes colour to red on the flowsheet topology. By inspecting information on this tab, the cause of the warning can be ascertained.

Stream tables

A stream table is a table containing information for on or more streams, such as composition, flowrate and temperature. Right-clicking on a stream connection, and selecting Create stream table, causes a table with results for that particular stream to be created at the bottom of the screen.

Creating a 'stream table'

 

Stream table

Hint

Multiple streams can be selected at the same time by holding the SHIFT key while selecting them. By using the 'Create stream table' menu option, while having a series of streams selected, a stream table for all these streams will be created.

Hint

A stream table, for all the streams in and out of a unit operation model, can be created by right-clicking the unit operation model and selecting 'Create stream table'. If the unit has multiple types of streams, care should be taken when selecting the type (e.g. 'gMLMaterial' for material streams) in the selector on the top left of the stream table view.

Stream analysers

Stream analysers are unit models that can be inserted into streams on the flowsheet to report additional results. These unit models have to be inserted into the streams prior to running the simulation to be able to obtain molar compositions and flowrates for that stream as well as additional desired properties such as phase fractions and compositions, densities and transport properties. There are stream analysers for the 'gMLMaterial', 'gMLMaterial_reversible' and 'gMLEnergy connection-types'. These unit are all contained in the 'gML Basics' library and can be included in a flowsheet model by dragging them from the library onto the flowsheet.

Example

Open the "Flash separation.gPJ-PB" example project and navigate to the flowsheet. Click on the connection (line) between valve "v1" and the sink "gas_product" and press the DELETE-key to delete it. From the "gML Basics" library drag a "Stream_analyser_gML" unit onto the flowsheet canvas, name it "sa1" and position it between valve "v1" and sink "gas_product". Connect its inlet to the outlet of valve "v1" and its outlet to the inlet of sink "gas_product".

Inserting a stream analyser ("sa1" ) into the gas product stream

Next, double-click on the stream analyser to open its dialog. From the dialog, select that vapour fraction is reported.

Specification dialog for stream analyser "sa1" . Vapour fraction is selected to be reported for this stream.

Now execute the simulation. After the simulation is completed, double-click on the stream analyser in the execution output window, and scroll down to note the vapour fraction for this stream.

Trajectory browser

All the results for the simulation are contained in the case file. By clicking on the folder 'Trajectories' in the case file, the values of all the variables contained in the flowsheet model can accessed. When the trajectories folder is expanded a list of all the units in the flowsheet model appears. The key variables for each unit can be found by expanding the folder for the individual unit models and then expanding the 'REP' folder that is inside. A list of all key variables for the unit model appears. Double-clicking on a variable opens a window in the work area with the results for that variable. By toggling between the 'Table' and 'Graph' tabs at the bottom of this window, the data can be represented both as (a series) of value(s) or as a graph.

Trajectories folder in the case file. For the unit model "NG_Feed" the variable "Average molecular weight" has been selected.

Example

After having previously opened the "Flash separation.gPJ-PB" example project previously and executed the process "Flash_separation_dynamic_initial_conditions", the project tree now contains a case file with all the results from this process. To determine what the outlet pressure of valve "V1" is at the start of the simulation, expand the 'Trajectories' folder in the case file, find and expand the "V1" folder, select the 'REP' folder and find the variable "Outlet pressure". Double-clicking this variable shows a table of simulation results.

Exporting results

Simulation results canbe exported from gPROMS ProcessBuilder for further processing. This may be of use for applying data analysis techniques or for presenting the data in a report.

Exporting by copy and pasting

The simplest way to export a limited amount of results is to simply copy and paste them from gPROMS ProcessBuilder into another application, such as, Microsoft Excel. Tables, for each variable in the trajectory browser, can be copy and pasted, by selecting a range of cells in them (or using CTRL+A to select all data for that variable) and using CTRL+C and CTRL+V in another application.

Data from stream tables can also be exported in the same way. Alternatively, an entire stream table can be copied by right-clicking in the stream table and selecting 'Copy stream table' from the context menu.

Copying stream tables

Exporting to CSV

When the results for a larger number of variables needs to be exported or the same variables need to be exported multiple times for multiple simulation runs of the same flowsheet model, it is more convenient to define data to export to a Comma Separated Value (CSV) file format. This file can then be opened by another application, such as Microsoft Excel, for further processing.

To export to CSV, right-click on the 'Trajectories' folder (or any folder contained within the 'Trajectories' folder), and select 'Export' from the context menu. Select 'Comma separated value (.csv) result file' in the menu that pops up.

Exporting results from the 'Trajectories' folder to CSV

A window is displayed, containing variables that can be selected, in a tree that corresponds to the contexts of the 'Trajectories' folder. By default every variable is selected. To export only a small subset of variables or variables for a particular set of units, first unselect the checkmark on the 'Trajectories' folder to remove all selections. Then, for each unit, select the desired variables.

Exporting the results for the vessel volume of separator "Drum".

When the desired variables have been selected, clicking the 'Export' button will cause these variables to be exported to a CSV file. The 'Save settings....' button can be used to save the current selection of variables. This is useful if more simulations, for the same flowsheet model, need to be run and for each run the same variables need to be exported. This selection can then be loaded and applied for a subsequent simulation by using the 'Load settings...' button.

The variables are exported to a CSV file in a stacked format. In MS Excel the PivotTable functionality can be used to construct 2 dimensional tables from the data.

Example

To export the results for the flowrates at the source and sinks from a simulation of the "Flash separation.gPJ-PB" example project, right-click on the Trajectories tab of the case file. Then, for the source model and each of the sink models, select only the "Mass flowrate" variables. Optionally, click "Save settings" to save the chosen selection for use in future simulations. Click on 'Export' and specify a CSV file.

Open Microsoft Excel and open the exported CSV file. Select all the data in the table with the 'VARIABLE', 'TIME' and 'VALUE' headers by clicking on the cell with 'VARIABLE', holding SHIFT+CTRL and pressing right and down arrow keys. Now from the ribbon menu select the 'Insert' and select the 'PivotTable' button to insert a pivot table.

Constructing a pivot table for the imported data

On the new worksheet created by the Pivot Table feature, select 'Variable', 'TIME' and 'Value'. Drag 'Variable' to the 'Column labels' area, 'TIME' to the 'Row labels' area and 'Value' to the 'Values' area. A two dimensional table will appear with the time indexing the rows and each variable contained in a separate column.
