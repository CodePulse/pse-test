Creating optimisations

To create a new Optimisation entity, right-click on the Project, select 'New Entity…' and set the Entity type to "Optimisation". It is important to give the optimisation the same name as the process to be optimised. Double-click on the new Optimisation to open and configure it.

Creating a new optimisation

The optimisation will open on the 'General' tab. Use the drop-down box to select a Process to optimise. This must be an existing Process in the same project. All valid Processes will appear in the drop-down list automatically. The objective can be to minimise or maximise any calculated variable in the process. Use the “Select” button to browse for calculated variables and select the objective. For Utility optimisation the objective is, usually, to minimise the total utility cost.

Add control variables by moving to the 'Controls' tab and clicking "Select". This brings up a browser of all flowsheet variables which are assigned in the specification dialog. Control variables must be assigned variables in the flowsheet, otherwise the optimiser is not able to control their value. Select controls to add by checking the box next to them. These will usually be the status and flow or steam/power production variables.

The variables will then appear in the 'Controls' tab. By default, controls are added as 'Continuous', with default initial guess and lower and upper bound values. Change these to provide sensible values for your problem.

Status variables should be added as 'Binary', with the initial guess value set to '1' unless the unit should only be used when absolutely necessary (for example, the lowest efficiency boiler). Flow variables should be added as 'Continuous', with the initial guess value set to that of the lower bound, unless a higher flow is much more likely.

Configuring the control variables

Finally, add constraints by selecting the "Constraints" tab. Constraints must be calculated variables in the flowsheet, and can either be Equality or Inequality end-point constraints. Usually, only Inequality constraints are needed, but Equality constraints can be added in a similar way. The "Select" button will bring up a browser from which calculated variables can be added as constraints in the same way as adding control variables. Each constrained variable must then be set a lower and upper bound.

The main Inequality constraints to consider are Calculated mass flow constraints, plus any Custom constraints defined for the problem. To define a positive constraint, set the lower bound to 0 and the upper bound to 1E30.

Configuring constraints

The "gPROMS Language" tab is automatically updated as settings are added to the optimisation, and can be viewed at any time to see the way controls and constraints are coded. The runtime solution writes this code automatically.
