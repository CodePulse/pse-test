User preferences

Many of gPROMS ModelBuilder's features can be customised by the user in a preferences dialog that is opened from the Edit menu. There are several preference categories, the main options for each are highlighted in the table below.

User preferences dialog

User preferences
Categories Available configurations and specifications
Workspace
  • Which action should be taken regarding the workspace at start-up:
    • open the last used
    • display a completely clean environment without any libraries or projects loaded
    • allow choosing a workspace.

  • The number of recently opened projects to appear at the bottom of the Project menu.

  • Whether to make backup projects when saving or not.
Entity editor

  • General text editor settings, such as font properties, tab size, thresholds for displaying numbers in scientific notation.

  • Syntax highlighting attributes.

  • Settings related to the auto-completion in gPROMS language.

  • Actions of the middle-mouse button.
View

  • Default mode of the editor (single or multiple windows).

  • The look and feel of gPROMS ModelBuilder. Three options are available: Metal, Windows1On the Windows platform, this is the default for fresh installations. It is recommended because it offers a better integration with the operating system, for instance the file dialogs. and Motif.

  • Individual gPROMS ModelBuilder messages may have been set not to be shown again, but this can be reset.

  • The default text size in the project tree should be overridden.

  • The text size to be displayed in the Project Tree.
Comparison

  • The colours used for text and group comparison.
File locations

  • The location of the temporary directory used by gPROMS ModelBuilder. This is the execution directory used to run model based activities.

  • The default starting location for the file dialog used for opening and saving projects.

  • The search path for any software components used by gPROMS, e.g. Foreign Objects, Foreign Processes etc.
Activity execution

  • Whether projects are saved prior to an activity execution.

  • The look and feel of the execution output window.

  • Defaults of the the Case configuration and execution control dialog.

  • Whether or not to show data points on graphs.

  • Whether graphs are shown in low-resolution mode.

  • The default number format.
Flowsheeting

  • Layout options of the topology view.

  • Flowsheet validation options.

  • Layout options of unit specification dialogs.
Schedule

  • The default width of schedule elements.
Licensed capabilities
  • Choice of optional, licensed capabilities gPROMS ModelBuilder should start with. If the chosen capabilities are enabled upon a restart, depends on the availability of their corresponding licenses. Should no licenses be available, for instance because all available ones are in use, gPROMS ModelBuilder will notify the user about this and allow to disable the capability request.
Advanced

  • Whether to display the interface language tab which offers direct access to XML code for the design of specification dialogs.

  • Configuration options related to backward compatibility of PSE's licensing.

    Warning

    Do not change any of the defaults unless instructed to do so by PSE support.
