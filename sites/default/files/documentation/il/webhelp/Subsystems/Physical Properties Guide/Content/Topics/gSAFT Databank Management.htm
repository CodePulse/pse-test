gSAFT Databank Management

Using a custom databank

The distributed databanks (VR_SW_databank.dtb and GC_Mie_databank.dtb) are provided in encrypted form, and their contents can neither be viewed nor modified. A separately licensed feature for databank management allows users to use the gSAFT foreign object with non-encrypted custom databanks. A non-encrypted databank can be used to initialise the gSAFT foreign object in the same fashion as an encrypted one.

gSAFT::non_encrypted_databank.xml<component1,component2,component3,…>

Note that the .xml extension is used for non-encrypted databanks.

When a custom databank is to be used for initialising the gSAFT foreign object, it needs to be attached to the project. This can be done by importing the file, by selecting Import files… from the Tools menu, as shown in the image below.

Compound definition

The definition of compounds in a VR_SW_Databank starts with the declaration of the <compounds> field as shown below.

Within the <compounds> field the following information needs to be provided:

  • The name of the compound
  • The molecular weight of the compound in units of [kg/kmol]
  • The coefficients for the ideal gas heat capacity of the compound. Four coefficients need to be defined so that the ideal gas heat capacity is calculated as a function of temperature based on the formula: .The heat capacity is calculated in units of [J / mol / K], with the temperature in units of [K].
  • The ideal gas enthalpy and entropy of formation of the compound in units of [J/mol] and [J/mol K], respectively
  • The SAFT-VR SW parameters of the compound:
    • The number of spheres (m parameter in SAFT-VR SW)
    • The segment diameter (σ [sigma] parameter in SAFT-VR SW)
    • The well-depth of the potential (ε/kB [epsilon] parameter in SAFT-VR SW)
    • The range of the interaction potential (λ [lambda] parameter in SAFT-VR SW)

    In the case of associating compounds the following additional information is required:

    • The number and type of each site
    • The energy of the association interaction (εHB/kB [crossEpsilon] parameter in SAFT-VR SW)
    • The range of the association interaction (rC [rc] parameter in SAFT-VR SW)

Selecting the equation of state

Group definition

The entire set of group types need to be declared inside the field <groups>, as shown below:

Each group entry requires the following information:

  • The name of the group
  • The molecular weight of the group in units of [kg/kmol]
  • The coefficients for the contribution of the group to the ideal gas heat capacity. Four coefficients need to be defined following the Joback group contribution method2 For more information on the expressions used and a table of the group-contribution method’s parameters see: https://en.wikipedia.org/wiki/Joback_method.. The heat capacity is calculated in units of [J / mol K], with the temperature in units of [K].
  • The coefficients for the contribution of the group to the ideal gas enthalpy and entropy of formation following the Joback method in units of [J/mol] and [J/mol K], respectively.
  • The SAFT-γ Mie parameters of the group:
    • The number of segments (ν* parameter in SAFT-γ Mie)
    • The shape factor (S [shapeFactor] parameter in SAFT-γ Mie)
    • The segment diameter (σ [sigma] parameter in SAFT-γ Mie)
    • The well-depth of the potential (ε/kB [epsilon] parameter in SAFT-γ Mie)
    • The attractive exponent of the interaction potential (λa [lambdaAttractive] parameter in SAFT-γ Mie)
    • The repulsive exponent of the interaction potential (λr [lambdaRepulsive] parameter in SAFT-γ Mie)

    In the case of associating groups the following additional information is required:

    • The number and type of each site
    • The energy of the association interaction (εHB/kB [crossEpsilon] parameter in SAFT-γ Mie)
    • The bonding volume of the association interaction (K [bondingVolume] parameter in SAFT-γ Mie)
