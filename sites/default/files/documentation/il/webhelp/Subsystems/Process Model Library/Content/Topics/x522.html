Material Ports

The PMLMaterial Connection Type is used to define the characteristics of a connection between two PML ports. It can be found in the PML Basics Library. For basic information on connectivity refer to the the Custom Modelling guide.

Connection Type Parameters

The PMLMaterial Connection Type declares two parameters:

  • no_components (INTEGER)

  • phys_prop (FOREIGN_OBJECT)

Each PML model groups its Material Ports into appropriate Port Sets to enable the Parameter values to propagate along the flow path; this enables the physical property parameters to be only specified in a single place - the material Source.

Connection type variables

The PMLMaterial Connection Type contains information and flow variables, namely:

Information variables

  • info_mass_fraction (no_components)

  • info_mass_specific_enthalpy

  • info_pressure

Flow variables

  • mass_flowrate

  • mass_fraction (no_components)

  • mass_specific_enthalpy

The information variables are used to describe the bulk fluid properties inside the Node at the point of connection. Whereas, the flow variables describe the properties of the material entering or exiting a model port. Therefore, depending on the direction of flow, the information mass_fraction may or may not be equal to the flow mass_fraction.

Flow reversal

The Connector model determines the flow direction:

  • If a reversible Connector model is used, the flow direction will be determined based on the pressure in the Node models either side of the Connector model.

  • If an irreversible1Most reversible Connector models can also be used as irreversible models. Typically a Case statement, without Switching, is used to distinguish between the reversible flow and the irreversible flow versions of a model. Connector model is used then the flow direction will always be in the nominal left-to-right direction.

If the flow direction is deemed left-to-right: the Connector model will equate the flow mass_fraction (x) and mass_specific enthalpy (h) to their information equivalents in the Connectorinlet port. The flow properties in the outlet port will be calculated from mass and energy balances over the Connector model.

On the other hand, if the flow direction is deemed right-to-left: the Connector model will equate the flow mass_fraction (x) and mass_specific enthalpy (h) to their information equivalents in the Connectoroutlet port. The flow properties in the inlet port will be calculated from mass and energy balances over the Connector model.

Port categories

It is worth noting that the above stream structure also permits Connector to Connector connections. On the other hand, Node to Node connections are not allowed. These rules are enforced using Port categories (see also: Declaring Connection Types). The Port categories that are defined for the PMLMaterial Connection Type enforce rules that only allow the model user to make valid connections; these are:
