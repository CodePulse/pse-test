Material property specification

Apart from process control Models all PML models contain a declaration of (at least) one physical property foreign object; typically defined as:

phys_prop AS FOREIGN_OBJECT "PhysProp"

To use the PML you only need to specify the Foreign Object to use for physical property calculations in Material Source dialogs (shown below): gPROMS will automatically propagate all relevant property parameters associated with the Source Unit (i.e. the number of components, foreign object etc.) along the flow path.

An example of such a foreign object class would be Multiflash or IPPFO. However, the PML can be used with any gPROMS-compliant physical property package including CAPE-OPEN enabled physical properties packages.

Note

If the physical property foreign object supports mass and molar basis – mass basis must be selected.

If using Multiflash the physical property parameter setting might look as shown below (refer to the examples in the gPROMS distribution for further details):

myunit.phys_prop := "Multiflash::mass:myfluid.mfl";

If multiple flow paths come into indirect contact with each other (e.g. in a heat exchanger), then gPROMS will simply use the properties of the mixture in each stream for the relevant energy balance calculations.

If multiple flow paths come into direct contact with each other, then it is vital that each stream has the same mixture specification (i.e. the type of components, their order, and the physical property calculation foreign object associated with each stream).
